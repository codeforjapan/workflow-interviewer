## 組織概要

| 項目 | 内容 |
|---|---|
| 正式名称 | 地域包括支援センター |
| 設置根拠 | 介護保険法 第115条の46（市町村設置義務） |
| 設置形態 | 市町村直営、または社会福祉法人・医療法人・社協等への委託 |
| 人員要件 | 保健師（または経験ある看護師）・社会福祉士・主任介護支援専門員の3職種配置 |
| 財源 | 介護保険特別会計（地域支援事業費） |
| 担当圏域 | 市区町村が設定する「日常生活圏域」ごとに1か所以上 |

### 4つの機能

| 機能 | 内容 |
|---|---|
| **総合相談支援** | 高齢者および家族からの相談を最初に受け止め、適切な支援につなぐ |
| **権利擁護** | 高齢者虐待の早期発見・対応、成年後見制度の活用支援 |
| **包括的・継続的ケアマネジメント支援** | 地域のケアマネジャーへのバックアップ、困難事例への対応 |
| **介護予防ケアマネジメント** | 要支援者・事業対象者の介護予防サービス計画作成 |

## 役割分担表

| 場面 | 市区町村の役割 | 地域包括支援センターの役割 | 連携の弱点 |
|---|---|---|---|
| 介護認定申請 | 認定審査会の運営・認定通知 | 申請の受理・代行・情報収集 | 認定結果の共有タイミングが遅い |
| 要支援者ケアプラン | 制度の枠組み設定・給付管理 | ケアプラン作成・サービス調整 | 計画変更時の市への連絡が属人的 |
| 高齢者虐待対応 | 最終的な措置権限（立入調査等） | 最初の相談受付・事実確認・調整 | 緊急性の判断基準が不明確 |
| 地域ケア会議 | 市主催の場合は事務局 | 個別ケースの情報提供・支援調整 | 会議の招集基準・参加機関が不定 |
| 成年後見制度 | 市長申立て（家族なき老人等） | 制度活用の相談対応・情報収集 | 申立て判断の基準が共有されていない |

## 構造的ギャップ

### ギャップ1: 市区町村とセンターの責任分界が曖昧（委託の場合）

委託型では「市の業務をセンターが代行している」が、委託仕様書に明記されない業務が多い。たとえば「複合困難ケースの最終責任は市か、センターか」が曖昧なまま運用される。虐待事案や生活保護申請につながるケースで判断が遅れる原因となる。

### ギャップ2: 圏域と実際のケース密度のミスマッチ

人口増加地域・高齢化が急速な地域でセンターのキャパシティを超えるケース数を抱えていても、圏域の見直し・センター増設が予算上後手に回る。「1センター当たりの担当高齢者数」の上限が形骸化しているケースがある。

### ギャップ3: 地域ケア会議（個別・政策）の混在

介護保険法は「個別地域ケア会議（個別ケースの支援調整）」と「政策立案のための地域ケア推進会議」を区別しているが、実務では混在して運用される。個人情報の保護水準・参加機関・目的が曖昧になりやすい。

### ギャップ4: センター間・センターと市の情報連携が属人的

複数センターがある市では、センター間のケース情報共有がメール・FAX・口頭に依存している。市（介護保険担当課）とセンターの情報共有システムがなく、「市が全体像を把握できていない」状態が生じる。担当者交代で引き継ぎが不完全になると、複数機関が並行して同一ケースに介入するか、誰も介入していない空白が生まれる。

## 連携のポイント

- 介護認定申請の受理代行はセンター経由が多い。認定結果の速やかな共有が要。
- 高齢者虐待の通報はセンターが第一報を受けることが多い。市への報告基準・タイミングを明文化しておく。
- 地域ケア会議の招集・議事録管理の責任者（市 or センター）を事前に確認する。
- 生活保護申請につながるケースは、センターから福祉事務所への橋渡しが重要。担当者間の顔の見える関係構築（→ 関係知識：内部KBで管理）。

# ステークホルダー (Stakeholders)

```yaml
folder_type: structural_knowledge
purpose: 自治体業務に関わる外部組織・機関の「制度上の役割と連携パターン」を記述
related_concepts:
  - journeys/         # 各ジャーニーで登場する組織の背景知識
  - workflows/        # 実際の業務実行時の協力機関
  - incident-catalog/ # 連携が機能しないときの穴と連鎖
```

> **このフォルダの位置づけ**
> ステークホルダーフォルダは **構造的知識（制度・一般的な役割）のみ** を扱う。
> 特定組織への関係的知識（担当者の顔・通りやすいタイミング・過去のトラブル）は
> **自治体の内部ナレッジベース** に記述する。この区別が重要。

---

## 構造的知識 vs 関係的知識

### 構造的知識（このフォルダに記述）

全国の自治体に共通する、**制度上・一般的に** 次のような情報：

- この組織の**法的根拠**は何か（例：社会福祉法 第109条）
- この組織の**組織属性**は何か（例：行政か民間か、非営利か営利か）
- この組織の**主な機能**は何か
- 自治体との**典型的な役割分担**は何か（表で整理）
- 連携が機能しないとき**何が起きるか**（ギャップ・インシデント）
- **この場面では連絡する** という典型的なトリガー

→ GitHub に置く。全国の自治体で参考にできる。改版可能（全国で利用される知識）

### 関係的知識（内部KBに記述）

特定自治体の、**この組織の担当者** に関する情報：

- 特定担当者の名前・内線番号・メールアドレス
- 「どのタイミングで連絡すると通りやすいか」（朝一番？月末？）
- 「この人は電話が苦手。メール必須」などの個人的な特性
- 過去のトラブル経緯（「去年の失敗で関係が冷えている」など）
- 個人的な人間関係や微妙な力学

→ 内部KBに記述。この自治体の固有情報として管理。改版不可（組織内で守る）。

## ステークホルダーの類型と整備状況

| 類型 | 説明 | 代表的な組織 | 整備状況 |
|---|---|---|---|
| **社会福祉・地域支援** | 住民の日常的な見守り・相談・支援を担う | 社会福祉協議会、民生委員・児童委員、地域包括支援センター、社協（災害VC） | ✅ 整備済み |
| **行政間連携** | 国・都道府県・他市区町村・広域団体・防衛機関 | 都道府県、日本年金機構、後期高齢者医療広域連合、福祉事務所、児童相談所、消防本部、自衛隊 | ✅ 整備済み |
| **行政内独立機関** | 首長から独立した行政委員会（同一自治体内） | 教育委員会、選挙管理委員会、監査委員 | ✅ 一部整備済み |
| **外郭団体・指定管理者** | 自治体が出資・設置する公社・財団・NPO | 福祉公社、自治会連合会 | 🔲 未整備 |
| **専門職・機関連携** | 医療・介護・教育など専門領域の機関 | ケアマネジャー、相談支援専門員、医療機関 | 🔲 未整備 |
| **調達・委託先** | 工事・システム・印刷などの商業契約先 | 建設業者、ITベンダー、印刷会社 | 🔲 未整備 |

## 整備済みファイル

### 社会福祉・地域支援

- **[shakyo-welfare-council.md](./shakyo-welfare-council.md)**
  社会福祉協議会（社協）の組織構造、自治体との役割分担、連携が機能しないときのギャップを記述。

- **[minseiiin.md](./minseiiin.md)**
  民生委員・児童委員の制度的位置づけ、担当地区と組織の仕組み、行政との情報流通パターンを記述。

- **[chiiki-houkatsu-shien-center.md](./chiiki-houkatsu-shien-center.md)**
  地域包括支援センターの4機能（総合相談・権利擁護・ケアマネ支援・介護予防）、市との責任分界、圏域問題を記述。

### 行政間連携

> **このカテゴリのポイント**
> 「行政間連携」ステークホルダーは、市区町村が単独では完結できない制度の「上位・並列・広域組織」を指す。
> **住民の窓口は市区町村でも、制度の決定権・実施責任が別の行政体にある**ことを可視化することが目的。

- **[prefecture.md](./prefecture.md)**
  都道府県の役割（生活保護の実施機関〈町村部〉、国保財政運営の責任主体、社会福祉法人認可等）を記述。
  市区町村と都道府県の役割境界で起きる典型的な混乱パターンを整理。

- **[japan-pension-service.md](./japan-pension-service.md)**
  日本年金機構の組織属性（特殊法人）と市区町村との役割分担を記述。
  死亡後の年金停止・未支給年金請求の案内漏れ、第3号被保険者の種別変更忘れ等のギャップを記述。

- **[kouki-koreisha-koiki-rengo.md](./kouki-koreisha-koiki-rengo.md)**
  後期高齢者医療広域連合の組織属性（特別地方公共団体）と市区町村との役割分担を記述。
  「保険者は広域連合、窓口は市区町村」という二層構造から生じる混乱パターンを整理。

- **[fukushi-jimusho.md](./fukushi-jimusho.md)**
  福祉事務所の2類型（都道府県設置型・市設置型）と生活保護実施責任の所在を記述。
  水際作戦・ケースワーカー過重負担・他機関との情報共有不備等のギャップを記述。

- **[jido-sodan-jo.md](./jido-sodan-jo.md)**
  児童相談所の設置者（都道府県・指定都市）と一時保護権限の独占、市区町村との役割分担（2022年法改正後）を記述。
  「通告したが動かない」認識ズレ・一時保護後の情報共有断絶・ヤングケアラーとの境界等のギャップを記述。

### 行政内独立機関

> **このカテゴリのポイント**
> 同じ自治体内にあるが首長から独立した「行政委員会」。
> 首長部局と独立した意思決定権限を持つため、**情報共有・連携に制度的な障壁**が生じやすい。

- **[kyoiku-iinkai.md](./kyoiku-iinkai.md)**
  教育委員会の独立性（地教行法）と首長部局との役割分担を記述。
  就学援助の実施主体の自治体差・スクールソーシャルワーカーの活用・ヤングケアラー支援での情報共有障壁等のギャップを記述。

### 災害対応・防災関連

- **[shobo-honbu.md](./shobo-honbu.md)**
  消防本部・消防署の組織属性（市区町村の機関）と災害対応での役割を記述。
  火災の罹災証明書（消防発行）と自然災害の罹災証明書（市区町村発行）の区別、救急搬送情報の市区町村への不共有等のギャップを記述。

- **[jieitai-saigai-haken.md](./jieitai-saigai-haken.md)**
  自衛隊の災害派遣の制度的位置づけ（国の機関・市区町村の指揮下にない）と要請フロー（都道府県知事経由）を記述。
  「市区町村長が直接要請できない」「自主派遣の条件」等のギャップを記述。

- **[shakyo-saigai-vc.md](./shakyo-saigai-vc.md)**
  社協が設置・運営する災害ボランティアセンターの機能・立ち上げタイミングと市区町村との役割分担を記述。
  情報共有の不足・VC機能の誤解・ボランティア急減と長期ニーズの乖離等のギャップを記述。

## 各ファイルの読み方

### 新しい自治体職員向け

1. **関連するジャーニー** を読む（例：困窮・複合課題を学ぶなら `journeys/poverty-and-crisis.md`）
2. **そのジャーニーで登場する組織** のステークホルダープロファイルを読む
3. **役割分担表** で「自分たちと何が違うのか」を確認
4. **ギャップ部分** で「これまで起きた落とし穴」を理解

### ベテラン職員向け

1. **連携が機能しないときのギャップ** から読む（実務の落とし穴に直結）
2. **連携ポイント** で「この場面では連絡する」の典型パターンを確認
3. 内部KBと組み合わせて、担当者の実名・タイミングを補充

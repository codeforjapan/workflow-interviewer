# gap-notes: 個人住民税

**このファイルの役割**:
標準仕様書の「あるべきフロー」と自治体の現実の差分を記録する。
差分はそのままFDE提案の根拠になる。

---

## 標準仕様書が定めているが、現実に機能していないこと

### ギャップ1：退職時の特別徴収から普通徴収への切替を住民が知らない

**仕様書の記述**: 退職者については、退職所得控除後の退職所得税を勤務先で一括徴収するか、
または翌年度以降に普通徴収（窓口・銀行納付）で徴収。住民の選択による。

**現実に起きていること**:
- 退職後の住民税が普通徴収に切り替わることを住民が認識していない
- 「退職時に最後の給与から天引きされたはず」と思い込み、翌年納期到来時に急に納付書が届いて驚く
- 「退職所得控除」と「普通徴収額」の仕組みが全く理解されていない
- 複数回退職（転職）した場合、どの退職所得控除を適用するかについて、住民と担当者の認識がズレる
- 退職金の受取時に「源泉徴収票」をもらっていない住民も多く、後年の申告時に困る

**FDE提案の根拠**:
仕様書は「退職所得税の計算方法」と「納付方法の選択肢」を定めるが、
「住民への事前案内」は定めない。これは典型的な「運用レベルの工夫」領域：
1. 退職予定者（会社から届出あり）に対する「退職後の税務手続きガイド」送付
2. 退職所得控除額のシミュレーション提示
3. 「来年から普通徴収が開始される」という明確な通知

これらの整備により、窓口での「突然の納付書に驚く」という問題は大幅に軽減可能。

---

### ギャップ2：転居時の課税自治体と居住自治体のずれ

**仕様書の記述**: 個人住民税の課税自治体は、1月1日現在の住所地（住民票所在地）。
転出入があっても、その年度の税額は1月1日現在の自治体から課税される。

**現実に起きていること**:
- 1月2日以降に転出した住民は、転出先の自治体から「納税通知書」を受け取り混乱
- 「もう転出したのに、前の自治体から納税通知書が来た」との苦情が増加
- 転入した側の自治体では「この人は1月1日現在いなかったから、課税対象外」という状態
- 3月に転出した住民は、転出先でなく転出元で納税通知書が発行されるため、
  「なぜ住んでいない自治体から納付書が来るのか」と理解困難
- 納付書の指定銀行が転出元のため、転入先の住民は納付に手間取る場合も

**FDE提案の根拠**:
仕様書は「1月1日が基準」という制度設計を定めるが、これは制度上の「谷間」であり、
自治体単独では解決不可（法律改正が必要）。しかし、市町村窓口での対応改善は可能：
1. 転出時に「今年度の納税通知書は転出元から来ます」という説明シートの整備
2. 「納付できるコンビニ・銀行」の拡大（全国どこからでも納付可能な体制）
3. オンライン納付（Pay-easy）の利用促進

これらの「説明と利便性向上」により、不満を軽減可能。

### ギャップ3：給与所得以外の収入（副業・不動産）の申告漏れ

**仕様書の記述**: 給与所得の他に副業収入・不動産所得・譲渡所得等がある場合、
住民税申告または確定申告の対象。給与のみの場合は申告不要。

**現実に起きていること**:
- 副業（フリーランス・アルバイト）収入を得ていても、給与天引きで「申告不要」と思い込み、
  申告せず、その後の税務調査で指摘されるケース増加
- テレワーク普及で副業が増加したが、「所得税と住民税は別」という認識がない住民が多い
- 不動産所得（親の家を貸している等）について「実はずっと申告していない」という相談が後年に
- YouTube配信やメルカリ販売などの少額所得を「申告するほどではない」と自己判断する住民が増加
- 「給与のみなら申告不要」という制度を過度に解釈し、医療費控除等の適用を受けない住民も多い

**FDE提案の根拠**:
仕様書は「申告対象者」の要件を定めるが「申告対象か否かの判定フロー」は定めない。
市町村の対応改善領域：
1. 給与天引き者向けの「確認チェックシート」（副業有無・不動産所得等）
2. 給与支払報告書の記載時に「その他所得の有無」欄を記載するよう事業者に依頼
3. 税務署の確定申告データとのマッチング時に「申告漏れ」を検出する定期レポート

これにより、事後的な問題ではなく事前に申告漏れを防止できる。

### ギャップ4：転職時の特別徴収切替がスムーズに行われない

**仕様書の記述**: 転職に伴い、転職元での最終給与で未徴収税を一括徴収するか、
または転職先での特別徴収に引き継ぐ。転職元から転職先への「転職所得控除」情報の移管が必須。

**現実に起きていること**:
- 転職時に「転職元で天引きしていない額」がいくらかを、転職先が把握していない
- 転職元から「転職届」が転職先に届かず、転職先では「新規採用者」として扱われることがある
- 結果として、転職元での未徴収分が普通徴収（市町村）で請求されるケースが多発
- 転職先でも新規に特別徴収が開始されるため、転職者は「二重納付」の状態に陥り、
  後年の調整で「還付」を受けるまで数か月の混乱が続く
- 給与支払報告書（1月31日期限）が転職先に遅れて到着すると、
  課税自治体での集計が遅延し、決定通知の送付が遅れる

**FDE提案の根拠**:
仕様書は「転職時の手続き」を定めるが「具体的なフロー」は定めない。
市町村は転職者の利便性向上のために以下を実装可能：
1. 転職予定者への「転職時の手続きガイド」の作成・配布
2. 「転職元での最終給与で調整」を推奨する説明
3. 転職先企業との連絡先登録（転職届の行き違いを防止）
4. 給与支払報告書の早期受付（12月末から受付開始）

これらにより、「転職時の混乱」を大幅に軽減できる。

## 標準仕様書が定めておらず、自治体が個別設計している領域

| 領域 | 内容 | 難易度 |
|---|---|---|
| 確定申告書の提出窓口 | 税務署のみか、市町村でも受け付けるか | 低 |
| 特別徴収対象外の判定基準 | 給与額の下限・退職予定者の対応等 | 中 |
| 給与支払報告書の提出方法 | 書面・電子申告（eLTAX）・その他 | 低 |
| 未納者への督促・対応 | 短期被保険者証発行の有無・連携方法 | 高 |
| 税額減免の申請受付 | 失業・災害等による減免の基準・フロー | 高 |
| オンライン納付システム | マイナポータル・Pay-easy等の導入レベル | 中 |

## 差分の構造的な意味

```
個人住民税の「混乱」は「給与・申告・転職」という複数の人生イベントが
同時に税務制度と接触する構造から生じている

  給与所得者の年間タイムラインの実際：
    1月   → 前年度の納税通知書到着（前の自治体から）
    1月以降 → 転職・異動が多い季節
    3月   → 確定申告期限。転職先との給与額が未確定のままの者も多い
    4月   → 新年度給与スタート。ただし前職での未納分の催告も同時期
    5月   → 給与から特別徴収開始。転職者は混乱の最中
    6月10日 → 決定通知書到着。転職者はこれが「確定」だと認識

  標準仕様が想定する「理想的なシーケンス」：
    1月1日現在の住所で課税 → 年度通し特別徴収 → 年度末調整で完納

  現実に発生する「現実的なシーケンス」：
    1月末までに転職 → 前の自治体での税額+新しい自治体での申告 → 混乱継続

この構造的なズレを完全に解決することはできない（法律改正が必要）が、
市町村レベルでできる改善は：

1. **タイムライン意識**：住民が転職時期に「税務手続き」を思い出すよう、
   複数のタイミングで段階的な案内を送付

2. **簡潔な説明**：「1月1日が基準」「来年から新しい自治体で課税される」など
   単純明快な表現で説明資料を統一

3. **利便性向上**：オンライン納付・コンビニ納付など、転出先からでも納付可能な体制

4. **事務処理の透明化**：「転職時の二重納付」が生じた場合の「還付プロセス」を
   あらかじめ説明し、「これは正常です」と安心させる

これらは「法律は変えられない」という制約の中で、
「市町村が主導的に運用を改善できる領域」として提案可能。
```

# gap-notes: 固定資産税

**このファイルの役割**:
標準仕様書の「あるべきフロー」と自治体の現実の差分を記録する。
差分はそのままFDE提案の根拠になる。

---

## 標準仕様書が定めているが、現実に機能していないこと

### ギャップ1：登記情報と課税台帳のタイムラグ（相続未登記問題）

**仕様書の記述**: 法務局登記情報と課税台帳を年1回マッチングし、所有権移転を反映。

**現実に起きていること**:
- 親が亡くなり相続人が決まっても、登記変更を放置する（登記費用負担）ケースが増加
- 登記されていない状態では、課税台帳は「亡くなった親」のままになる
- 翌年度の納税通知書が故人宛で送付され、相続人が対応に困る
- 複数の相続人がいる場合、誰が納税義務者か不明確なまま督促が行われることもある
- 相続人が複数いて共有状態でも、登記がなければ課税台帳上は1人のままという矛盾

**FDE提案の根拠**:
仕様書は「登記情報と連携する」を定めるが「登記がない場合の対応」は定めない。
これは法制度上の課題（相続登記の義務化は2024年から）だが、市町村窓口での対応フローとして：
1. 相続人調査票の様式統一
2. 法務局への定期的な「未登記物件」問い合わせ
3. 税務署の相続税申告情報の活用

などは自治体が主導的に設計可能な領域。

---

### ギャップ2：評価替え時の住民への説明負担

**仕様書の記述**: 3年に1回、全資産の評価額を再計算し、評価額が変わることを住民に通知。

**現実に起きていること**:
- 評価替え年度（例：2022年）に「評価額が上がった」という連絡を受けた住民から
  窓口・電話に苦情が殺到する
- 「前年度より値上がりした」「説明がわからない」という問い合わせが通常の10倍以上に
- 説明会を開催しても参加率は低く、窓口での個別説明が対応の大半
- 「なぜ評価額が上がるのか」「この金額は正しいか」を丁寧に説明するのに
  1件当たり20〜30分要することも珍しくない
- 説明会用のスライドやチラシは各自治体で異なり、わかりやすさのばらつきが大きい

**FDE提案の根拠**:
仕様書は「評価額の計算方法」を定めるが「説明の方法・わかりやすさ」は定めない。
この対応は自治体の裁量領域であり、FDE介入で統一できる典型的な課題：
1. 評価額の変化を視覚的に示す「個別説明シート」の標準様式
2. 評価替えの仕組みを説明する「住民向けガイドビデオ」
3. オンラインシミュレーター（自分の物件がどう変わるか確認可能）

### ギャップ3：農地転用後の地目変更遅延による過少課税

**仕様書の記述**: 農地転用許可後、地目を「田」「畑」から「宅地」等に変更し、評価額を上げて課税。

**現実に起きていること**:
- 農地転用許可を取得しても、所有者が課税台帳の地目変更申告をしないケースが多い
- 課税担当者が農業委員会の転用許可情報を確認していない自治体では、
  何年も「田」のままで課税されることがある
- その結果、実際には宅地なのに田んぼ価格で課税され、過少課税状態が続く
- 3年後の評価替えまで気づかれず、その時点で突然「評価額が大幅上昇」として住民が驚く
- 転用許可から数年後に住宅建設が完了した場合、その間は「地目：田」「用途：宅地相当」
  という矛盾がシステム上発生することもある

**FDE提案の根拠**:
仕様書は「農業委員会と連携して地目変更」を定めるが「定期的な確認フロー」は定めない。
これは単なる「情報連携の漏れ」であり、以下の対応で解決可能：
1. 農業委員会からの月次「転用許可情報」を自動受信し、マッチング
2. 不一致（許可あるのに課税台帳に反映されていない）の場合、自動で異議申立レポート
3. 担当者の確認・決定フロー（職権変更 vs. 所有者申告待ち）

### ギャップ4：新築住宅軽減措置の申告漏れ

**仕様書の記述**: 新築住宅は3年間（長期優良認定で5年間）、家屋税額の1/2を軽減。
ただし、建築年の翌年度から適用（当初は満額課税）。

**現実に起きていること**:
- 新築完了の翌年度から軽減が適用されることを所有者が知らず、
  「去年納税したのに軽減されていない」と不満を持つ
- 新築住宅の軽減制度自体を住民が認識していないケースも多く、
  本来なら軽減対象なのに「誰も申告していない」状態
- 建築確認申請・完了検査との連携がないと、課税担当者も軽減要件を把握できない
- 母屋のみ新築軽減を受け、その後増改築した部分は軽減対象外という複雑さが説明困難
- 長期優良住宅認定の情報が課税担当者に届いていない場合もある

**FDE提案の根拠**:
仕様書は「軽減措置の要件」を定めるが「住民への事前案内」は定めない。
新築物件の所有者に対して、建築完了時点で送付される標準的な
「軽減措置のお知らせ」と「申請方法」の整備が、窓口問い合わせを大幅に削減できる。
また、建築管理システムとの連携により、自動判定・手動確認のプロセス化も可能。

## 標準仕様書が定めておらず、自治体が個別設計している領域

| 領域 | 内容 | 難易度 |
|---|---|---|
| 新築・増改築調査の実施フロー | 完了検査と連動するか、別途所有者申告か | 中 |
| 相続未登記物件への対応 | 故人名義のまま課税継続 vs. 相続人調査して変更 | 高 |
| 農地転用情報の定期確認 | 農業委員会との情報交換方法・頻度 | 中 |
| 評価替え説明会の開催方法 | オンライン・集団・個別など形式の選択 | 低 |
| 評価額異議申立への対応 | 聞き取り・書類提出の受け付け方法・期間 | 中 |
| 境界確認・筆境界情報の活用 | 法務局提供の筆境界データをどう利用するか | 高 |

---

## 差分の構造的な意味

```
固定資産税の「複雑さ」は多数の外部システムとの連携不完全性に起因する

  理想的なフロー：
    法務局登記 → 自動で課税台帳反映
    農業委員会許可 → 自動で地目変更
    建築完了検査 → 自動で新築軽減判定
    3年後 → 自動で評価替え・説明資料生成

  現実：
    登記情報と課税台帳が「別システム」のため月次対照作業が必須
    農業委員会との情報交換が手作業（FAX・メール）
    建築完了から課税台帳登録までにタイムラグ（さらに登記遅延で倍増）
    評価替え時の説明会開催が「年間で最も時間を要する業務」に

この構造的な課題の解決は「技術的連携」（API化・リアルタイム同期）ではなく
「運用の最適化」（定期確認フロー・自動マッチング・標準説明資料）として提案できる。

特に「説明負担の集中」を緩和するために：
- 事前案内の充実（評価替え前3か月から段階的に実施）
- わかりやすい個別説明シートの標準様式化
- 評価額シミュレーターのオンライン提供

など、住民との接点を明確にした改善が、職員の時間割き問題を大きく改善する。
```

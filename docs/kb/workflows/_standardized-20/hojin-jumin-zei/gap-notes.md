# gap-notes: 法人住民税

**このファイルの役割**:
標準仕様書の「あるべきフロー」と自治体の現実の差分を記録する。
差分はそのままFDE提案の根拠になる。

---

## 標準仕様書が定めているが、現実に機能していないこと

### ギャップ1：本店・支店の按分計算における担当者ばらつき

**仕様書の記述**: 本店・支店等が複数都道府県に所在する場合、従業員数・給与額・売上高等に基づいて按分計算した比率で各地域に税額配分

**現実に起きていること**:
- 按分の計算ロジックは仕様書に示されるが、実装時の詳細判断（どの期間の従業員数か、異動タイミングの扱い等）が各自治体で異なる
- 同じ法人が複数市区町村から申告を受けた場合、自治体によって算定された税額が異なることがある
- 担当者によって「最後に更新された支店データを使う」「1月1日時点のデータを使う」など判断がばらつく
- 支店数が多い法人では二重課税や過度な減免が発生することがある

**FDE提案の根拠**:
仕様書は「按分すること」を定めるが「いつの時点でのデータを用いるか」は定めていない。
担当者向けの「支店按分計算の標準フロー」と、市区町村間での情報共有（特に支店移動時）を
FDE介入で整備できる典型的な領域。
法人からの修正申告請求や苦情が発生している実態がある。

---

### ギャップ2：法人設立・本店移転届出の遅延と消失

**仕様書の記述**: 法人設立や本店移転は法務局に登記された時点で課税対象となり、自治体は登記情報または法人からの届出に基づいて課税台帳を更新

**現実に起きていること**:
- 法人設立後、登記からが遅れ、税務申告（確定申告）が来て初めて自治体が法人の存在を知ることが多い
- 本店移転届出が申告書に記載されるまで、当該自治体の課税台帳に登録されない
- 資本金が低い小規模法人ほど申告の遅延が著しく、均等割が「未登録」のまま数年経つケースがある
- 法務局との連携は完全ではなく、各自治体が登記情報を主動的に入手していない実態がある

**FDE提案の根拠**:
これは制度設計の「谷間」。法務局への登記情報システムへのアクセスが全自治体で可能なわけではなく、
現状は申告書に頼らざるを得ない。法人設立時の「税務届出案内」の強化と、
法人マスタの「登録待機状態」管理が FDE で整備できる領域。

→ 関連業務: `workflows/_standardized-20/hojin-jumin-zei/flow-standard.md`（設立届フロー）

---

### ギャップ3：休眠法人の均等割賦課継続問題

**仕様書の記述**: 法人が休業・廃業した場合、届出に基づいて課税を停止

**現実に起きていること**:
- 休業届を出さない休眠法人に対し、自治体が継続して均等割を賦課し続けるケースが多い
- 届出を受けても「いつまで休業するのか」が不明確で、翌年度も課税する判断がされることがある
- 廃業予定の法人も届出せず放置され、結果として「不在法人」への課税が溜まることがある
- 徴収不能額が増加する一方、法人本体との連絡手段がなく回収困難

**FDE提案の根拠**:
仕様書は「届出に基づいて停止」を定めるが、届出がない限り賦課が自動継続される。
税務署や法務局の登記情報から「休廃業情報」を入手し、課税停止の根拠とする
自動判定フローと、書面での確認フロー（例：法人への「税務申告ない企業への調査通知」）が
FDE介入で整備できる領域。

---

### ギャップ4：申告書の電子申告率の低さと紙処理の残存

**仕様書の記述**: 申告書の電子申告機能を実装することで効率化

**現実に起きていること**:
- 小規模法人では紙申告が大多数で、電子申告率は地域によって20～40%程度に留まる
- 紙申告書の入力・スキャン・保管が資源集約的で、担当者の業務量が多い
- 申告書到達のタイミングがばらばらで、「申告期限内に税額が確定しない」状況がしばしば発生
- 電子申告と紙申告で提出データのフォーマット差異があり、システム上の統一的な処理が難しい

**FDE提案の根拠**:
電子申告の推進自体は仕様書の範囲だが、「中小法人への案内」「代理人申告の簡素化」
「申告期限内の確定を優先する催促体制」など、運用面での FDE 介入が必要な領域。
マイナンバー連携強化による事前確認データの提供も有効。

## 標準仕様書が定めておらず、自治体が個別設計している領域

| 領域 | 内容 | 難易度 |
|---|---|---|
| 支店按分の実装ルール詳細 | 従業員数のカウント方法・データ取得タイミングの明確化 | 高 |
| 法人設立から申告までの待機期間管理 | 登記後未申告法人の催促フロー・登記情報自動取得の可否 | 高 |
| 休廃業法人への確認・回収フロー | 課税停止判定の基準・税務署データの活用可能性 | 中 |
| 複数自治体への分割課税時の調整 | 複数市区町村への説明・二重課税防止の仕組み | 中 |
| 中小法人への電子申告推進 | インセンティブ設計・簡易申告制度の創設 | 低 |

## 差分の構造的な意味

```
法人住民税の「穴」は税収逸失と行政負荷の両面で深刻化する

  法人設立・移転の未届出
    ↓ 課税台帳への登録遅延
    ↓ 税額の適正化遅延
    ↓ 均等割・法人税割の賦課漏れ
    ↓ 複数年にわたる未納が発生
    ↓ 徴収困難化・不納欠損増加
       ↑ここで初めて「対応」が必要になる

また、本店・支店の按分計算がばらつくと：
  - 法人側：「他の市区町村の賦課と矛盾」と苦情
  - 自治体側：「説明が難しい」と苦情処理に時間
  - 市区町村間：「どちらが正しいか」を判断できない

これらを断ち切る介入点は：
1. 法人設立時の税務届出・登記情報確認の強化
2. 本店・支店の按分ロジックの全国統一（法務省・総務省による仕様化）
3. 小規模法人への電子申告インセンティブ設計

gap-notes の情報は、その介入点を設計するための根拠になる。
```

## 法人住民税における法令と実装のズレ

地方税法で定める法人住民税の課税要件は全国統一だが、実装する自治体の課税事務取扱は以下の点で分散している：

- **支店按分の計算方法**：仕様書に記載されているが、詳細な実装は自治体に委ねられている
- **申告期限の定義**：法人税申告期限から2か月以内が原則だが、地域によって運用が異なる
- **電子申告システム対応**：全自治体で実装されているわけではない（小規模市区町村では未対応も）
- **法務局登記情報との連携**：システム的な自動連携ができている自治体は限定的

これらのズレは FDE（フロー・ディスカウント・エンジニアリング）のターゲット領域となり得る。

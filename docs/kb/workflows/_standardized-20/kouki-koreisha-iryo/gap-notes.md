# gap-notes: 後期高齢者医療

**このファイルの役割**:
標準仕様書の「あるべきフロー」と自治体の現実の差分を記録する。
差分はそのままFDE提案の根拠になる。

---

## 標準仕様書が定めているが、現実に機能していないこと

### ギャップ1：75歳誕生日前後の保険証の切れ目

**仕様書の記述**: 誕生日前日に国保喪失、誕生日当日に後期高齢者医療の資格取得。保険証は誕生日までに到着予定。

**現実に起きていること**:
- 広域連合から送付される後期高齢者医療保険証が誕生日までに到着しないケースが多い（郵送遅延・住所誤記）
- その間に受診が必要な場合、国保も後期高齢者医療も資格がない状態になる可能性
- 住民が誕生日以降も旧国保保険証を使用してしまい、レセプト返戻・請求トラブルが発生
- 当番医・夜間救急での対応が曖昧（資格確認システムのタイムラグで確認できない）

**FDE提案の根拠**:
仕様書は「保険証発行」を定めるが「到着時期の保証」は定めない。この問題は：
1. 広域連合への事前連絡（早期発送依頼）
2. マイナカード・オンライン資格確認の積極活用
3. 保険証未到着時の「一時資格証明書」発行フロー

の整備により、介入可能。

→ 関連業務: `workflows/_standardized-20/kokumin-kenko-hoken/gap-notes.md`（ギャップ1：住基連携のタイムラグ）

---

### ギャップ2：後期高齢者医療広域連合との役割分担の曖昧さ

**仕様書の記述**: 市町村と広域連合は協力して制度を運営。市町村は資格申請・窓口対応、広域連合は保険料決定・徴収。

**現実に起きていること**:
- 保険料決定に異議がある場合、市町村窓口で「広域連合に聞いてください」と言われ、住民が混乱
- 保険料減免の申請を市町村窓口で受け付けても、決定は広域連合であり、市町村が結果を伝えるだけ
- 特別徴収への切替基準（住民税のコード連携）が市町村と広域連合で認識ズレ
- 未納者への督促・短期被保険者証発行が広域連合主導のため、市町村は対応内容を事後報告で知る
- 転出・死亡時の処理が市町村と広域連合で連携遅延することがある

**FDE提案の根拠**:
これは制度設計上の「階層構造」であり、仕様書の空白ではなく「境界曖昧性」の問題。
市町村窓口での**対応フロー・説明スクリプト**と**広域連合との定期情報交換会議**の設置が、
自治体内の混乱を解消する介入点。

### ギャップ3：特別徴収（年金天引き）の仕組みを住民が理解していない

**仕様書の記述**: 老齢年金受給者は年金から特別徴収（天引き）される。普通徴収から特別徴収への切替は、前年の住民税特別徴収状況に基づき自動判定。

**現実に起きていること**:
- 75歳到達者が「なぜ年金から勝手に引かれるのか」と窓口に苦情を持ち込む
- 「普通徴収になると思っていた」「申請が必要だと思っていた」との認識ズレ
- 保険料決定通知書に「特別徴収対象者に切り替わります」と表記されていても、
  その後の納付方法の実際の流れ（初回は普通徴収、4月から特別徴収）を住民が理解できない
- 初回納付書を受け取って「どうして2通来るのか」と問い合わせが増加

**FDE提案の根拠**:
仕様書は「特別徴収すること」を定めるが「説明・案内」は定めない。
75歳誕生日の6か月前から段階的に送付される案内資料の**わかりやすさ向上**
（図解・シミュレーション例示）は、各自治体が整備すべき領域として提案可能。

---

### ギャップ4：限度額適用認定証申請の必要性を住民が知らない

**仕様書の記述**: 自己負担限度額を超える場合、限度額適用認定証を事前に提示することで、病院窓口での一時負担を軽減。

**現実に起きていること**:
- 多くの住民が「後期高齢者医療に加入すれば高額医療費は自動的に還付される」と思っている
- 限度額適用認定証の申請が必要であることを知らず、病院で窓口全額払いした後、申請して数か月後に還付される道を選ぶ
- 低所得世帯で「高額医療費」として還付手続きを取ってくれると期待しているが、
  実際には「認定証がないと窓口負担が大きい」という逆転現象が発生
- 入院予定が判明した段階で初めて申請する住民が多く、窓口での急な申請対応が負担

**FDE提案の根拠**:
仕様書の「高額療養費制度」と「限度額適用認定証」は別制度であり、その違いを
住民が理解するためには、保険証と同時に送付される**"申請を勧める通知"**や
**病院掲示用ポスター**の整備が必要。これは標準仕様の周知・啓発領域。

→ 関連インシデント: 高額医療費の事後還付と事前軽減の混乱については、
   後期高齢者医療広域連合のウェブサイト・住民向けリーフレットの統一化提案も可能。

## 標準仕様書が定めておらず、自治体が個別設計している領域

| 領域 | 内容 | 難易度 |
|---|---|---|
| 75歳到達の事前案内スケジュール | 何か月前から何をどのようなタイミングで送付するか | 中 |
| 保険証未到着時の対応フロー | 一時資格証明書の発行基準・有効期限 | 高 |
| 広域連合との情報交換方法 | 担当者間の連絡体制・定期会議の開催頻度 | 中 |
| 限度額認定証の申請受付方法 | 郵送・窓口・オンラインの選択肢 | 低 |
| 未納者への対応フロー | 市町村による催告・相談と広域連合の短期被保険者証発行の連携 | 高 |
| 転出・転入時の処理フロー | 他自治体への情報提供・受け取りのタイミング | 中 |

## 差分の構造的な意味

```
後期高齢者医療制度の「複雑さ」は階層構造と制度設計の非対称性にある

  住民の視点：
    「75歳になると自動的に後期高齢者医療に切り替わる」
    「医療費は自動的に計算・還付される」
    → 実際：資格は自動だが、認定証申請・納付方法選択・特別徴収への理解は自分で

  自治体の視点：
    「資格取得・喪失は自動化できた」
    「保険料決定・徴収は広域連合がする」
    「でも窓口では全ての質問に答えなければならない」
    → 広域連合の決定・方針に異議がある場合の対応が曖昧

この構造的な齟齬を解消する介入点は：
1. 住民向けの段階的・わかりやすい事前案内
2. 市町村窓口での統一的な説明スクリプト
3. 広域連合との定期的な情報交換・ルール確認

これらはいずれも「標準仕様の周知・徹底」ではなく「実装レベルの工夫」として提案できる領域。
```

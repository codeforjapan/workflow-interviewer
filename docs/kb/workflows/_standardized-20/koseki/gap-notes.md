# gap-notes: 戸籍・戸籍の附票

**このファイルの役割**:
標準仕様書の「あるべきフロー」と自治体の現実の差分を記録する。
差分はそのままFDE提案の根拠になる。

---

## 標準仕様書が定めているが、現実に機能していないこと

### ギャップ1：本籍地と届出地が異なる場合の書類受付・転送処理

**仕説書の記述**: 本籍地と異なる場所で届出を受け付けた場合、本籍地の市区町村へ書類を転送し、本籍地で戸籍に記載する

**現実に起きていること**:
- 婚姻届など、本籍地以外での受付が可能だが、転送先の本籍地での処理に数日間の遅延が発生
- 住民は「受付した＝完了」と認識するが、本籍地での確認・記載完了まで数日待つ必要がある旨が説明されていないことが多い
- 本籍地が遠い場合（例：北海道の本籍地、東京での届出）、書類の郵送に1週間要することもあり、「いつ完了するか」不明確
- 転送書類が到着しない、紛失するなどのトラブルが時々発生するが、責任の所在が曖昧
- 異なる自治体の窓口では「いつまで待つべきか」についての説明スクリプトが統一されていない

**FDE提案の根拠**:
仕説書は「転送すること」を定めるが、転送のタイミング（同日か翌営業日か）、追跡方法、処理完了の確認フローが定義されていない。
「届出から完了までの標準日数」を明示し、異なる自治体間での処理進捗情報共有の仕組みを
FDE で整備できる典型的な領域。特に転入地と本籍地が遠く離れているケースへの対応が重要。

→ 関連業務: `workflows/_standardized-20/jyumin-ido/gap-notes.md`（転出入届との連携）

### ギャップ2：夜間・休日受付した届出の翌営業日処理と遡及効の説明

**仕説書の記述**: 日中受付できない場合、夜間・休日の書類受付を行い、翌営業日に処理完了

**現実に起きていること**:
- 一部の自治体は夜間・休日の当番制を敷いているが、全国的には限定的
- 金曜日の夜間に受け付けた出生届は「月曜日に処理」となり、「遡及は出生日か受付日か」が曖昧に説明される
- 死亡届の場合、翌営業日処理では喪主が「いつから相続人として扱われるのか」という質問に対して、確実な説明ができない職員が多い
- 休日に届出を受け付けた場合の「収受印」の時刻をどう扱うか（実時刻か翌日の日付を記載するか）が自治体で異なる
- 住民は「夜間に持ってきたのに翌日扱いなのか」と納得しないことがある

**FDE提案の根拠**:
仕説書は夜間・休日受付の「機能」を定めるが、「効力発生日」「遡及の範囲」「当番者の確認手続き」などの運用ルールが定義されていない。
「夜間受付した届出の効力は受付日付となる」という法的解釈を踏まえた上で、
住民向けの説明スクリプト、相続関係の事前説明（死亡届の場合）などが FDE で整備できる領域。

### ギャップ3：令和6年施行の戸籍法改正（本籍地外での謄本取得）の運用定着

**仕説書の記述**: 令和6年3月27日から、全国の市区町村で本籍地以外の場所でも謄本・抄本を取得できる

**現実に起きていること**:
- システム対応が完了していない自治体がまだ存在し、「本籍地でのみ交付」と案内している窓口がある
- 戸籍情報連携システム（法務省）の導入が予定より遅れている自治体が多く、仮システムでの対応が継続
- 住民は「全国どこでも取得できる」と認識しているが、実際には自治体によって機能差がある
- 本籍地外での交付申請を受け付けても、本籍地の戸籍番号確認のために照会が必要で、その処理時間が見通せない
- 代行取得（申請者本人が来庁しない郵送申請）との組み合わせで、認証手続きの負荷が増加している

**FDE提案の根拠**:
これは新制度の「定着期」における実装の課題。仕説書は「機能提供」を定めるが、各自治体の準備状況がばらつき、
住民サービスの「ばらつき」が発生している。「全市区町村で同じレベルの機能提供」を実現するための
システム統合スケジュール、段階的対応ルール、代行申請時の確認フロー等が FDE で整備できる領域。

→ 関連施策: 戸籍情報連携システム（法務省推進）のロールアウト加速

### ギャップ4：外国人との国際結婚・離婚の手続き複雑さ

**仕説書の記述**: 外国人が婚姻・離婚する場合、その国の法律に基づいた手続きが必要な場合があり、自治体は本国領事館との確認手続きを行う

**現実に起きていること**:
- 外国人との婚姻届を受け付ける際、本国法律との適合性確認が必要だが、領事館の確認に数週間要することがある
- 配偶者の国籍が「不安定な国」「紛争地」の場合、書類確認が困難で、「仮受付」で対応するしかないケースがある
- 離婚届の場合、本国法が「協議離婚」を認めないこともあり、日本での協議離婚届が受け付けられないことがある
- 担当者の経験と知識に依存する部分が大きく、複数市区町村で手続きが異なることがある
- 外国人住民の増加に伴い、英語以外の言語での説明を求められることが増えているが、体制が整備されていない

**FDE提案の根拠**:
仕説書は「確認すること」を定めるが、具体的な確認方法（領事館への照会、本国法の解釈等）は自治体の自助努力に任せられている。
外務省の「国別法律情報」や多言語翻訳サービスとの連携、領事館照会の標準フロー、
「即日は難しいケース」の事前説明スクリプト等が FDE で整備できる領域。
多文化共生推進部局との連携体制の整備も必要。

## 標準仕様書が定めておらず、自治体が個別設計している領域

| 領域 | 内容 | 難易度 |
|---|---|---|
| 夜間・休日当番制の整備 | 死亡届等の緊急受付体制・翌営業日処理フロー | 高 |
| 本籍地外での謄本交付の対応進度 | 戸籍情報連携システム導入スケジュール・仮対応ルール | 高 |
| 外国人婚姻・離婚の手続き簡素化 | 本国法律確認フロー・多言語対応・領事館連携 | 高 |
| 離婚後の親権・養育費の説明 | 協議離婚届とは別の民法上の手続き案内 | 中 |
| 戸籍の附票の住所履歴の活用 | 過去の住所確認が必要な事案への対応（遺産分割等） | 中 |
| 届出不受理制度の運用 | DV被害者等の本籍地変更時の保護措置 | 中 |

## 差分の構造的な意味

```
戸籍業務の「穴」は、届出人の「どこまで待つべきか不明確」という不安感につながる

  本籍地と届出地が異なる届出（婚姻等）
    ↓ 書類が本籍地に転送される（数日要する）
    ↓ 本籍地での処理（書類受理・記載）
    ↓ 完了までの日数が不明確なまま届出人は帰宅
    ↓ 「いつ完了するのか」「証明書はいつ取得できるか」問い合わせ増加
    ↓ 窓口負荷増加

また、外国人との婚姻が増えるとともに：
  - 本国法律の確認に時間がかかる
  - 領事館の対応が不確実
  - 受け付けられない届出もある
    ↑ 住民側の納得感の低下

新制度（令和6年改正）の運用定着が遅れると：
  - 「全国どこでも取得可能」という広告と現実のズレ
  - 自治体によって利用可能な機能が異なる
  - システムトラブル時の代替手段が不明確
    ↑ 住民の不信感につながる

これらを断ち切る介入点は：
1. 本籍地転送時の「標準処理日数」の明示と進捗情報共有
2. 夜間・休日受付の全国統一体制（最低限、大都市での整備）
3. 戸籍情報連携システムの完全導入と全自治体での機能統一
4. 外国人婚姻の事前相談体制・領事館照会の自動化
5. 新制度対応の「実装チェックリスト」提供

gap-notes の情報は、その介入点を設計するための根拠になる。
```

## 戸籍法改正（令和6年）に伴う運用の変化

令和6年3月27日施行の戸籍法改正により、以下が大きく変わった：

| 改正内容 | 旧制度 | 新制度 |
|---|---|---|
| 謄本・抄本の取得場所 | 本籍地の市区町村のみ | 全国どこの市区町村でも可（戸籍情報連携システム経由） |
| 本人確認の方法 | 窓口で身分証提示 | マイナカード確認など電子化も可 |
| 代行取得 | 戸籍謄本などで親族証明が必要 | 委任状で簡素化される（制度側の方針） |
| 情報セキュリティ | 各自治体独立管理 | 法務省の戸籍情報連携システムに一元化 |

このシステム移行期における「対応ばらつき」が、現在の市区町村間での機能差につながっている。

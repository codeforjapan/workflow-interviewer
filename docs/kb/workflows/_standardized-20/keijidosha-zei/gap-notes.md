# gap-notes: 軽自動車税（種別割・環境性能割）

**このファイルの役割**:
標準仕様書の「あるべきフロー」と自治体の現実の差分を記録する。
差分はそのままFDE提案の根拠になる。

---

## 標準仕様書が定めているが、現実に機能していないこと

### ギャップ1：廃車手続きを完了しないまま乗り続ける車両への対応

**仕様書の記述**: 軽自動車検査協会への廃車申告・登録により、市区町村は自動通知を受けて課税台帳から削除

**現実に起きていること**:
- 廃車手続きをせず乗り続けている車両が多数存在（特に過疎地域）
- 軽検協から廃車通知が来ないまま、自治体は課税を継続する
- 旧所有者への継続課税により、納税通知が届かない住所地への不納欠損が増加
- 「昔買った軽自動車をずっと乗っている」「手続きが面倒」という住民の声は存在するが、実態把握が困難
- 廃車申告を促すための案内が充分でなく、制度周知が不足している

**FDE提案の根拠**:
仕様書は「廃車通知を受けたら削除する」という受動的なフローのみを定める。
市区町村が主動的に「使用されていない車両」を特定し、廃車を促すフロー（例：複数年未納の軽自動車への確認通知）が
FDE で整備できる領域。軽検協との連携タイムラグの改善も重要。

---

### ギャップ2：転居に伴う課税自治体の変更遅れ

**仕説書の記述**: 軽自動車検査協会への転入・移転登録により、市区町村間での課税地変更が自動処理される

**現実に起きていること**:
- 軽検協との連携がバッチ処理のため、転入・転出から課税データの更新まで3～7日間のタイムラグが発生
- 転出予定者が転出前に軽自動車税納税通知を受け取ることがあり、「払うべきか払わないべきか」が混乱
- 転入後に旧住所地から納税通知が来るケースもあり、住民からの問い合わせが多い
- 軽検協の登録データと市区町村の課税台帳の「所有者住所」がズレることで、納付忘れが増加
- 転居と同時に軽自動車の登録変更をしない住民が存在（「税務上の住所」と「登録住所」の分離）

**FDE提案の根拠**:
連携のリアルタイム化は仕様書の範囲外だが、連携遅延を前提とした「中間状態管理」（転出予定者への説明、転出後の催促フロー等）が
FDE で整備できる領域。軽検協との情報連携の加速化も並行して推進が必要。

---

### ギャップ3：バイク（原付）の廃車申告の放置

**仕説書の記述**: 軽自動車と同じく原付バイク（125cc以下の原動機付自転車）も登録・廃車手続きが必要

**現実に起きていること**:
- 原付バイクの廃車申告は市区町村の税務課で受け付けるが、軽自動車と異なり検査機構がない
- 住民が「壊れたから乗らない」という場合でも廃車手続きの存在を知らず、課税が続く
- 相続時に原付の名義が切り替わらず、被相続人名義での課税が継続する事例が多い
- 課税台帳の「使用停止」と「廃車」の区別が曖昧な自治体が多く、判断ルールが不統一
- 引越しと同時に廃車申告をするケースが少なく、旧住所地からの納税通知が「宛先不明」で戻ってくる

**FDE提案の根拠**:
原付バイクは軽自動車ほど「検査協会連携」が制度化されていないため、完全に「自治体の自助努力」に依存している。
「原付バイク廃車申告の案内」強化（引越し時、相続時）と「使用停止フロー」の整備が
FDE で実施できる領域。マイナンバーとの連携で相続情報自動取得も可能性がある。

### ギャップ4：身体障害者等の減免申請の毎年繰り返し

**仕説書の記述**: 身体障害者等が所有・使用する軽自動車の減免申請に基づいて、100%または75%の減免を適用

**現実に起きていること**:
- 減免の有効期間が「当年度のみ」と定めている自治体が多く、毎年度の減免申請が必要
- 住民は「去年減免してもらったのに、今年も申請が必要？」と疑問を持つが、運用ルール説明が不足している
- 減免申請書の様式が複雑で、必要な添付書類が自治体によって異なる
- 減免期間切れに気づかず申請忘れ→急に課税→納付を求められるという苦情が発生
- 身体障害者のほか、生活保護世帯・母子世帯の減免要件を独自に設定している自治体があり、案内ばらつきがある

**FDE提案の根拠**:
減免要件の設定は各自治体の条例に委ねられているが、一度減免対象と認定された者に対して
「複数年の減免継続申請」や「自動更新」の仕組みを導入することで、住民の手続き負荷を大幅に軽減できる。
これは仕様書の「空白領域」であり、FDE で「標準的な減免管理フロー」を提案できる領域。

---

## 標準仕様書が定めておらず、自治体が個別設計している領域

| 領域 | 内容 | 難易度 |
|---|---|---|
| 廃車手続き未了車両への督促フロー | 複数年未納・登録情報乖離の早期発見と廃車促進 | 高 |
| 転居時の課税地自動変更ルール | 軽検協との連携遅延を前提とした中間状態管理 | 高 |
| 原付バイク廃車申告の案内・相続時対応 | 引越し・相続時の申告促進・自動取得の検討 | 中 |
| 身体障害者等の減免申請の簡素化 | 複数年減免・自動継続・添付書類の標準化 | 中 |
| 納税通知書のわかりやすさ向上 | 初年度特例・環境性能割との区分記載 | 低 |

## 差分の構造的な意味

```
軽自動車税の「穴」は微細だが累積すると大きな逸失につながる

  廃車手続き未了・転居時の登録変更遅延
    ↓ 課税台帳と軽検協データの乖離
    ↓ 旧住所地への納税通知が「宛先不明」で戻る
    ↓ 納付忘れ・不納欠損化
    ↓ 小額だが多数の滞納処分案件発生
       ↑ 市町村の徴収事務の負荷増加

また、原付バイクの廃車申告が制度化されていないため：
  - 相続時に被相続人名義のまま課税継続
  - 使用停止と廃車の区別が曖昧
  - 検査協会連携がないため「お知らせ」のチャネルがない

これらを断ち切る介入点は：
1. 廃車促進キャンペーン（「乗らない車は廃車を」という周知）
2. 軽検協との連携の加速化（リアルタイム化、エラー通知強化）
3. 原付バイク廃車の「案内チャネル」整備（引越し・相続時）
4. 減免申請の「複数年自動継続」仕組みの導入

gap-notes の情報は、その介入点を設計するための根拠になる。
```

## 軽自動車税における原付バイクと軽自動車のシステム差異

軽自動車と原付バイク（原動機付自転車）は同じ「軽自動車税」の対象だが、登録・廃車のシステムが異なる：

| 項目 | 軽自動車 | 原付バイク |
|---|---|---|
| 登録場所 | 軽自動車検査協会 | 市区町村役場（税務課） |
| 廃車手続き | 軽検協 | 市区町村役場 |
| 市区町村への通知 | 自動連携（軽検協から） | 住民からの申告による |
| 他市区町村への転入手続き | 自動処理（検査協会が統括） | 転出元での廃車申告+転入地での登録申告が必要 |
| 登録データの正確性 | 比較的高い（検査機構を通すため） | 低い（自治体任せ） |

このシステム差異が、原付バイクでの手続き遅延や廃車放置の温床となっている。

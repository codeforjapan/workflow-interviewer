# gap-notes: 軽微発注・納品検査

**このファイルの役割**:
「全国的な典型例」と実際の自治体の運用の差分を記録する。
軽微発注は毎日のように行われる業務であるため、差分も顕著に表れやすい。

---

## 典型例と現実の主要なギャップ

### ギャップ1：随意契約の分割発注（脱法的運用）

**典型例で想定している運用**:
- 100万円（物品）・250万円（工事）が限度額
- 限度額を超える発注は「競争入札」が必須
- 同じ品目を複数に分けて限度額以下にすることはNG

**現実に起きていること**:
- 「春に50万円、秋に50万円」と分割発注している
- 「A課で50万円、B課で50万円」と別の課に分けて発注
- 「年間100万円の予算だから、限度額ギリギリまで1社に発注できる」と毎年同じ業者に
- システムやツール導入時に「10万円ずつ×10回」に分割して随意契約で調達
- 工事で「修繕工事（その1）¥250万」「修繕工事（その2）¥250万」と2件に分割

**なぜこうなる**:
- 「分割発注は禁止」というルールが周知されていない
- 「同一施設・同一用途なら別だろう」という勝手な判断
- 「競争入札のプロセスが面倒」「小額なら急ぐから競争入札できない」という理由
- 過去から「この業者とはずっとこの金額で発注してきた」という慣例がある

**リスク**:
- 会計検査・監査で指摘される（「違法な随意契約」と判定される可能性）
- 競争機会を奪っている（他社が参入できない）
- 発注透明性の問題（説明責任を問われる）

**FDE/業務改善の観点**:
- **「限度額を超える場合は入札」というルールを明確化・周知**
  - 研修・ポスター・チェックリスト
- **発注額の累計を自動計算するシステム**
  - 「この業者に今年度いくら発注済みか」を可視化
  - 限度額に近づいたら警告
- **分割発注の事前審査**
  - 複数件の関連発注がある場合、「これ1件で発注すべきでは」と事前にチェック
- **監査・検査での定期確認**
  - 分割発注の実態を年1回確認・改善

### ギャップ2：納品検査の形骸化

**典型例で想定している運用**:
- 納品時に開梱・内容確認 → 検査成績書に記載 → 「OK」と署名
- 不良品なら「NG」と記載して返品・交換依頼

**現実に起きていること**:
- 「検査」と称して、見た目ちょっと確認するだけ
- 「○個来るはずだから、段ボール箱の個数で OK」
- 「いつもこの業者だし、品質は大丈夫」と確認せずに「OK」
- 検査成績書に署名 → 実は中身を確認していない
- 後で「あ、色が違ってる」「1個足りない」と気づいても、「検査済み」扱いで手遅れ
- 役務（サービス）の場合、「清掃してくれたらしいから OK」と実地確認なし

**なぜこうなる**:
- 「検査は手続きだから、押印すればいい」という理解不足
- 時間がない（他の仕事に追われて、納品検査まで手が回らない）
- 「業者との関係が悪くなるから、いちいちNGとは言えない」という心理的抵抗
- 検査者の教育・引き継ぎがない（「検査とは何か」を新人に説明していない）

**問題点**:
- 不良品を引き取らされたまま → 市がロス
- 不具合が後で見つかって、対応が面倒
- 「検査済み」なのに品質問題がある → 信用失墜

**FDE/業務改善の観点**:
- **検査成績書のフォーマット改善**
  - 「仕様通りか（色・サイズ等）」「数量確認」「品質確認」のチェックボックスを明示
  - 1個チェックしないと「保存できない」という仕組み
- **検査者の研修**
  - 「検査とは何か」「よくある不良品のパターン」を定期教育
  - 役務検査のポイント（清掃業務なら「床・壁・天井のどこを確認するか」）
- **不良品の記録**
  - 「この業者からは不良品が多い」という記録を蓄積
  - 次の発注時に「見積は安いけど、品質に問題がある」と判断材料に
- **サンプル検査の導入**
  - 大量納品の場合、全数確認ではなく「ランダムに◎個確認する」ルール化

### ギャップ3：見積合わせの記録が残らない

**典型例で想定している運用**:
- 複数社に見積依頼 → 見積書を回答受領 → 比較表を作成 → 「なぜこの業者か」を記録
- 後で「なぜこの業者を選んだのか」と聞かれても「A社が最安値」と説明可能

**現実に起きていること**:
- 「◎◎商社に電話して見積もらった」「その場で『いくら？』『◎◎円です』『OK、発注して』」
- 見積書は受け取るがファイルしない / どこかに置き忘れた
- 「B社はいくらだった？」と聞かれて「えっと…忘れました」
- 後から「なぜこの業者？」と聞かれて、根拠が説明できない
- 複数社に見積依頼したはずなのに、「結局は前回と同じ業者」という不透明感

**なぜこうなる**:
- 「少額案件だし、見積記録は要らないだろう」という判断
- 見積書をもらった後、保存場所が決まっていない
- 口頭での見積（電話）を記録しない習慣

**問題点**:
- 監査・検査で「見積は取ったのか」と聞かれて、証拠がない
- 「最安値で選んだ」と言えない（記録がないから）
- 同じ品目でも、次の発注時に前回の金額を参考にできない

**FDE/業務改善の観点**:
- **見積記録の「テンプレート」を用意**
  - 見積日 / 見積依頼先 / 品名 / 金額 / 納期 / 担当者
  - 紙でもExcelでも、必ず記録する
- **見積書の保存義務を明記**
  - 「発注金額 ◎万円以上は見積書を必ずファイル」というルール
- **見積比較表の作成ルール化**
  - 2社以上に見積依頼した場合、比較表を作成 → 「なぜこの業者か」を記入
- **口頭見積の記録**
  - 「電話で◎社から聞いた」という場合でも、メモ（日付・金額・担当者名）を残す

### ギャップ4：発注から検査まで単一担当者（牽制機能がない）

**典型例で想定している運用**:
- 見積取得：「発注担当者」
- 発注：「発注担当者」
- 納品検査：「別の担当者」または「複数で確認」
- 支払：「会計部門」
- 監視：監査・検査部門（年1回）

**現実に起きていること**:
- 「◎◎課で10万円の物品購入」→ 同じ担当者が「見積→発注→受け取り→検査→OKサイン」を全部やる
- 「この業者から買う」と決めたら、見積を取らず → 「見積はこの金額」と記録 → 発注
- 納品検査も「自分で確認した」と言い張る（実は十分に確認していない）
- 不透明な発注が起きやすい（「本当に最安値？」と第三者がチェックしない）

**なぜこうなる**:
- 小額案件が大量にあり、いちいち別の人に見てもらう時間がない
- 小額なら「チェックは不要」という勝手な判断
- 組織が小さく、複数の目を入れられない

**問題点**:
- 「自分に都合のいい業者を選ぶ」ことができてしまう（人情的に付き合いがある業者等）
- 不良品納入の際、「自分で発注して、自分で検査OK、自分で支払」→ 誰も気づかない
- 架空発注（実際には納品されていないのに「納品受け取り→OK」と記録）のリスク

**FDE/業務改善の観点**:
- **発注と検査の分離**
  - 「自分が発注したものは、自分以外が検査する」ルール
  - 小額でも、最低限このルールは守る
- **見積取得の複数化**
  - 少額でも、最低2社から見積依頼（「見積1社」は避ける）
- **検査のチェックリスト化**
  - 何をどこまで確認するかを明記 → 確認漏れを防ぐ
- **監査の強化**
  - 月単位での「小額発注」抽出 → ランダムに監査
  - 「分割発注していないか」「検査は本当にしているか」を確認

## 典型的な問題シーン

### シーン1：「これ色が違ってます」と納品後に気づいた場合

```
【典型例では】
→ 納品時に「色合い確認」をして、その場で「OK」or「NG」判定
→ OK → そのまま使用 or NG → 返品・交換

【現実に起きる流れ】
→ 「開梱した。◎◎が¥X,XXX で来ました。OK押します」
→ 1週間後、実際に使うときに「あ、この色前と違う」
→ 「検査は終わってますから…」と業者に言われる
→ 返品できず、そのまま使用するか、追加購入の予算から差し引く
→ 損失
```

---

### シーン2：「見積はいくらだった？」と聞かれた場合

```
【典型例では】
→ 「◎◎会社 ¥50,000、◎◎商社 ¥55,000、◎◎店 ¥60,000
    比較の結果、◎◎会社が最安値で選定」と即答できる
→ 見積書ファイルを出せる

【現実に起きる流れ】
→ 「あ、電話でもらった気がするんですけど…」
→ 「見積書？どこに置いたっけ…」
→ 「多分◎◎会社が一番安かったと思う」と曖昧に答える
→ 監査「記録がありませんね」と指摘
```

---

### シーン3：「この工事、来年度も同じ工事を◎社に発注してた」という場合

```
【典型例では】
→ 「工事その1」と「工事その2」の間に、
   十分に期間があれば別件として扱える
→ ただし「同じ工事を◎社に継続」という実態は記録される
→ 「毎年同じ業者ばかり」と監査で指摘される可能性も

【現実に起きる流れ】
→ 去年A社に「修繕工事（その1）250万円」
→ 今年またA社に「修繕工事（その2）250万円」
→ 実は同じ内容の工事を2年連続で同じ業者
→ 「競争入札すべきでは」と指摘される
→ 「毎年やってたので」という言い訳は通用しない
```

## 業務改善の入口

### すぐに取り組める施策

1. **見積記録の様式を決める**
   - 「見積依頼記録」（日付、依頼先、金額）を紙またはExcelで
   - 最低限「3者の見積金額を並べる」ようにする

2. **検査成績書を改善**
   - チェックボックス式にして「確認した項目」を明示
   - 「この欄に全てチェック入るまで押印不可」という運用

3. **発注と検査の分離ルール**
   - 少なくとも「小額案件でも、発注者と検査者は別」に
   - 難しければ「◎万円以上は必ず2人でチェック」

### 中期的な改善（システム企画の観点）

- 軽微発注システム（見積→発注→納品検査→支払）の整備
- 見積記録・比較の自動化
- 発注額の累計表示（限度額チェック）
- 不良納品の記録蓄積（業者評価に反映）
- 随意契約の「脱法分割」を自動検出

## 記録例

### 見積記録表

| 見積日 | 品目 | 見積依頼先 | 金額 | 納期 | 担当者 | 選定理由 |
|---|---|---|---|---|---|---|
| 2026-04-02 | コピー用紙 A4 20冊 | ◎◎商社 | ¥9,000 | 4/10 | 鈴木 | A社 ¥9,000 / B社 ¥9,500 / C社 ¥10,000 → A社最安値 |
| 2026-04-03 | プリンタ用紙 50冊 | ■■オフィス | ¥15,000 | 4/15 | 佐藤 | 1社のみ（品種特殊） |

### 検査成績書（フォーマット改善版）

```
【納品検査記録】

発注部門：企画課
品目：コピー用紙（A4、白色、500枚/冊）
業者：◎◎オフィス用品店
納品日：2026年4月10日

【確認項目】
□ 品名・型番が正しいか  ✓ OK
□ 数量が正しいか（20冊）   ✓ OK （段ボール開梱、1冊ずつ確認）
□ 品質に問題ないか        ✓ OK （破損・汚損なし）
□ 納期内か               ✓ OK （指定日4/10に納入）

【総合判定】
  ✓ OK（納品を受け取り）
  □ NG（返品・交換要求）
  □ 条件付OK（軽微な不具合あり、使用可）

【検査者】
企画課 □□
確認日：2026年4月10日
署名：□□ ██████

【不具合記録（NGs の場合のみ）】
（該当なし）
```

## 分割発注チェックリスト

```
□ この業者への今年度の発注累計はいくらか
□ 限度額（物品100万/工事250万）に達していないか
□ 複数に分割した発注がないか
  →「◎年度◎月に◎万円」「◎年度◎月に◎万円」と分割していないか
□ 関連する複数案件を1件にまとめられないか
□ 年度内に同じ品目を複数回購入していないか
  →「春と秋で同じ物品を別発注」していないか
```

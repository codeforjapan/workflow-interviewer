# 外部会議ロジスティクス — ギャップノート

このドキュメントは、「外部会議ロジスティクス（審議会・有識者会議・ケース会議）」の**理想的なフロー**（flow-canonical.md）と**実際の自治体実務**の間にある4つの重要なギャップを記述します。

---

## ギャップ 1: 欠席委員の意見処理の未規定

### 現象

| 理想フロー | 実務現状 |
|----------|--------|
| 開催通知時に「欠席時の意見聴取方法」を明示。書面意見、事前質問受付などの規定がある | 欠席委員の意見聴取を想定していない、または規定が曖昧。「ご欠席のため議事録をご確認いただき、ご意見あればお知らせください」という事後的な対応になる |
| 定足数要件を条例に明記。法定数以上の出席が必要な場合は、会議成立条件を確認 | 定足数の確認が形式的。「欠席者が多かったが会議を開催した」「事後的に欠席委員からの意見が反映された」というケースが発生 |

### ギャップの発生メカニズム

1. **欠席の常態化**
   - 委員の多忙、遠方開催、代理参加の制限などにより、欠席率が20～30%に達する会議がある
   - 特にケース会議では、関係機関の都合がつきにくく、参加機関の欠席が相次ぐ場合がある

2. **意見聴取のプロセス未設計**
   - 開催通知時に「欠席時の書面意見受付」を明示しない
   - 当日議事録作成後に、欠席者へ「ご確認ください」と送付するのみ
   - 事前質問制度がない自治体がほとんど

3. **定足数の認識のあいまい性**
   - 条例に「定足数＝出席者の過半数」と記載されていても、「延べ招集人員の半数」なのか「実出席者の半数」なのか不明確
   - 「全委員の2分の1以上」という規定も、欠席者が多い場合の扱いが曖昧
   - ケース会議では定足数規定そのものが存在しないケースがほとんど

### 影響・リスク

- **決定の正当性を問われるリスク**: 十分な人数の参加なしに重要決定が下された場合、異議が生じる可能性
- **重要意見の喪失**: 専門的知見をもつ委員が欠席し、その意見が事後的に反映されない、または軽く扱われる
- **ケース会議での支援漏れ**: 関連機関の欠席により、必要な情報共有・支援が実施されない

### 改善のポイント

| 改善案 | 対象会議 | 実装難度 |
|-------|--------|--------|
| 開催通知時に「書面意見受付」の期限を明示。欠席者からの意見書を配布資料に含める | 審議会・有識者会議 | 中 |
| ケース会議の参加機関に対して、「代理参加またはメール意見」の提出を事前に依頼 | ケース会議 | 低 |
| 条例・要綱に「定足数＝実出席者（書面意見提出者を含む）の過半数」と明記 | 全会議 | 低 |
| 欠席者の多い会議については、定期的に「参加方法の改善」を検討する仕組みを設置 | 全会議 | 中 |

## ギャップ 2: 議事録確定プロセスの属人性と遅延

### 現象

| 理想フロー | 実務現状 |
|----------|--------|
| 「委員確認→修正→確定」の各段階に期限を設定。例：委員確認は10営業日、確定は5営業日以内 | 確定までの期間が規定されていない。会議から3ヶ月～1年後に公開されるケースも多い |
| 議事録確定の責任者・判断基準が明確。どのような修正で最終確認が必要かが定められている | 「主任が確認する」「課長の決裁を得る」など、個人の負担に依存。担当者が不在になると処理が止まる |

### ギャップの発生メカニズム

1. **期限設定の欠如**
   - 議事録作成・確認・確定の各段階に「いつまでに」という期限がない
   - 「すぐに公開する」という義務認識が弱い
   - 他業務に追われ、議事録確認が後回しになる傾向

2. **確認プロセスの無限ループ**
   - 「委員確認→修正→他の委員との調整→再確認」という反復が発生
   - 複数委員から矛盾する修正意見が出た場合、調整に時間がかかる
   - 「確定基準」が不明確なため、「これ以上修正はない」という判断ができない

3. **担当者の不在対応**
   - 異動・病休などで確認者が不在になると、処理が止まる
   - 代替者が不明確で、未確定議事録が積まれたままになる
   - 数年前の会議の議事録が「まだ確定していない」という事態が発生

4. **公開の義務化の不十分さ**
   - 法令で「議事録を作成する」と定められているが、「公開する」「いつまでに」は明記されていない
   - 情報公開請求されるまで公開されないケースも

### 影響・リスク

- **透明性の低下**: 決定が下されても、その過程が外部に見えない
- **説明責任の放棄**: 決定内容に対する異議が生じた場合、その根拠となる議事録が参照できない
- **事業の引き継ぎ支障**: 担当者交代時に「前任者が進めた決定の背景」が不明確
- **コンプライアンスリスク**: 公開情報管理（公文書管理法）との齟齬が生じる可能性

### 改善のポイント

| 改善案 | 対象会議 | 実装難度 |
|-------|--------|--------|
| 議事録作成→委員確認→確定の各段階に明確な期限を設定。例：委員確認は会議から2週間以内 | 全会議 | 低 |
| 「修正の分類基準」を策定。「事実関係の誤り」は対応、「表現の修正」は軽微として一定期間後に自動確定 | 審議会 | 中 |
| 確定権者を明記。例：「主任が修正を反映し、課長が確定」と役割分離 | 全会議 | 低 |
| 3ヶ月以上未確定の議事録については、「修正受付を終了し確定する」という自動確定ルールを導入 | 全会議 | 中 |
| 公開システムへの自動掲載を制度化。確定後、翌営業日に自治体ウェブで公開 | 全会議 | 中 |

## ギャップ 3: 利益相反（COI）管理の形式化

### 現象

| 理想フロー | 実務現状 |
|----------|--------|
| 委員選定時にCOI宣言書を提出させ、「同一業界」「親会社・子会社」などの関係者を除外 | 「同業者でなければOK」「明らかな利害関係がなければ大丈夫」という緩い運用 |
| 会議議事の中で「特定企業の利益にかかわる議題」がある場合、利害関係委員は退席・不投票 | 議事中のCOI退席ルールが明文化されていない。「利害関係者が列席しながら議論に参加」というケースがある |

### ギャップの発生メカニズム

1. **COI宣言の形式化**
   - 委員候補から提出させるCOI宣言書が、簡潔な項目しかない
   - 例：「金融機関の利益に関わる審議会で、金融機関の営業職員が委員」など、関連の深さが評価されていない
   - 「別の子会社には属していないので大丈夫」という浅い判断
   - 聞き取り調査をしない自治体もある

2. **議事進行時のCOI確認の欠如**
   - 委員名簿段階で「一般的なCOI」を確認するのみ
   - 議題が具体的に示されるまで、利害関係の深さが分からない
   - 「この議題はA企業に有利/不利」という判定が、会議当日まで先送りされる
   - 事務局が「これはCOI退席が必要では」と指摘しない

3. **退席ルールの曖昧性**
   - 条例・要綱に「利害関係がある場合は退席」と書かれていても、「利害関係」の定義が不明確
   - 「企業の方針が反映される可能性がある全ての議題」なのか、「直接の経営判断に影響する議題」なのか
   - 委員自身が「退席が必要」と判断する基準がない

4. **ケース会議での多機関参加による隠れたCOI**
   - 地域ケア会議で、福祉サービス事業者が「支援方針決定」に参加する場合のCOI管理がない
   - 「自社サービスの利用を推奨する可能性」が明示的に管理されていない

### 影響・リスク

- **決定の公正性を問われるリスク**: 利害関係者が参加した審議会の決定が「不公正」と異議を唱えられる可能性
- **後発的な利益相反の発覚**: 決定後に「実は委員が利害関係者だった」という事が発覚
- **ケース会議での支援の偏り**: 自社サービスの推奨が、客観的支援方針より優先される可能性

### 改善のポイント

| 改善案 | 対象会議 | 実装難度 |
|-------|--------|--------|
| COI宣言書を詳細化。「出身企業」「出身団体」「現在の給与源」「実績・契約先」を明記 | 全会議 | 低 |
| 委員選定段階で「本審議会の議論が対象企業の経営に影響するか」を事務局で判定 | 全会議 | 中 |
| 議事進行時に「本議題でのCOI退席者」を事前に明示。会議冒頭で「○○委員は第3議題で退席」と告知 | 全会議 | 低 |
| ケース会議では、参加機関ごとに「自社サービス関連議題での発言制限」を明記 | ケース会議 | 中 |
| COI判定ガイドを自治体として策定。「直接的利害関係」「間接的利害関係」の基準を示す | 全会議 | 中 |

## ギャップ 4: ケース会議の参加機関・招集基準の不定

### 現象

| 理想フロー | 実務現状 |
|----------|--------|
| 「どのようなケースには、どの機関を招集するか」の基準を明記。例：児童虐待で医療機関の関与がある→小児科医師を招集 | 「どの機関を呼ぶか」は個別案件ごとに主催部局の判断。多くの場合、定期メンバーのみが参加し、必要な機関が漏れるリスクがある |
| 招集ルールを定期的に見直す仕組みがある。過去の事例から「この機関も必要だった」という学習が反映される | 招集基準が固定化。「毎回同じ機関」「誰かが指摘するまで変わらない」という運用 |

### ギャップの発生メカニズム

1. **招集基準の欠如**
   - 要保護児童対策地域協議会（児童福祉法25条2項）や地域ケア会議の場合、「構成機関」は法令に定められている
   - しかし「個別ケース会議で、実際にどの機関を招集するか」については規定されていない
   - 例：「児童虐待ケースであっても、医療機関が関与していなければ医師を呼ばない」「学校との連携の有無で学校管理職の出席を判断する」など

2. **主催部局による判断の属人性**
   - 「児童福祉課が開催する個別ケース会議」の場合、児童福祉課長が参加機関を決める
   - 新任の課長が引き継いだ場合、「定期的に参加している機関」に自動的に案内するのみ
   - ケース固有の状況（複合課題、新規関係者など）を踏まえた判断がされない

3. **必要機関の漏れ**
   - 児童虐待ケースで「警察の関与の可能性」が見落とされる
   - 高齢者ケースで「成年後見人」の参加がなく、財産・身上監護の課題が未検討
   - 複合課題（児童×経済困窮、高齢者×介護）で、両分野の専門機関が揃わない

4. **事後的な「漏れ」の発見**
   - 会議実施後に、「あの機関も呼ぶべきだった」という指摘が出ても、既に決定されている
   - 次回開催時に「前回の不十分さ」を踏まえた構成にする、という対応になる

### 影響・リスク

- **支援の空白**: 必要な機関との連携がないため、支援が一面的になる。結果として虐待の再発、高齢者の孤立化などのリスク
- **情報の分断**: 複数の関係機関が個別にケースに関わっているが、情報共有の場に揃わない
- **法定義務の未履行**: 要保護児童対策地域協議会として定足数や構成が要件に達していない場合、実質的には「協議会として機能していない」

### 改善のポイント

| 改善案 | 対象会議 | 実装難度 |
|-------|--------|--------|
| 「招集基準」を文書化。例：児童虐待ケース→児童相談所・学校・保健センター・警察（必要時）を標準メンバーとする | ケース会議 | 中 |
| ケースの「複合課題マトリクス」を作成。「児童＋経済困窮」「高齢者＋認知症＋一人暮らし」など、各パターンで必要機関を定義 | ケース会議 | 高 |
| 個別ケース会議の開催前に「参加機関リスト」を事務局で作成し、主任が確認する仕組みを導入 | ケース会議 | 低 |
| 年1回、「過去のケース会議で『呼ぶべき機関が漏れた』という事例」を総括し、招集基準を更新 | ケース会議 | 低 |
| 定期的に「隣接分野の関係機関」を参加させ、「自分たちの役割」と「他機関との役割分担」を認識させるトレーニング | ケース会議 | 中 |

## 関連ステークホルダー・組織

### 内部ステークホルダー

- **主管課（審議会担当）**: 各部局の審議会・会議の企画責任者
- **事務局・庶務担当**: 日程調整、資料作成、記録業務
- **情報政策部門**: 議事録の公開管理、情報セキュリティ確認
- **法務・コンプライアンス部門**: COI管理ガイドライン、秘密会議のルール管理

### 外部ステークホルダー

- **委員・有識者**: 会議参加者、意見提出者
- **関係機関（ケース会議）**: 児童相談所、学校、医療機関、警察、福祉事業者など
- **市民・市議会**: 決定結果・議事録を通じて知る利害関係者
- **報道機関**: 公開情報としての議事録・決定事項の報道

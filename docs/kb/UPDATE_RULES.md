# 更新ルール

標準仕様書の改版・法改正・国通知が発生した際に、  
どのファイルをどう更新するかの判断基準を定める。

---

## 変更のトリアージ（4象限）

変更内容を受け取ったとき、まず以下の4象限に分類する。

|  | flow-standard 変更あり | flow-standard 変更なし |
|---|---|---|
| **gap-notes 変更あり** | **A象限**: フロー構造・判断ルールが変わる（重大改訂） | **C象限**: フローは変わらず新たな実務判断が発生（運用追加） |
| **gap-notes 変更なし** | **B象限**: フロー構造は変わるが判断ルールは同じ（軽微改訂） | **D象限**: バージョン番号・リンクの更新のみ |

**分類例:**
- R6.10改正（高校生年代拡大） → **A象限**（支給額算定ロジック・対象年齢が変わる）
- 児童手当2万円補正予算給付 → **C象限**（通常フローは変わらず、事務処理が追加）
- 標準仕様書のバージョン番号更新のみ → **D象限**

---

## 優先度の基準

| 優先度 | 対象 | 期限 |
|---|---|---|
| **即時** | インシデントリスクに関わる変更 / 住民への不利益に直結する変更 | 当日中 |
| **高** | A象限の変更（フロー構造の変更） | 1週間以内 |
| **中** | B・C象限の変更 | 次回定期更新時 |
| **低** | D象限（バージョン番号・リンク更新） | 月次確認時 |

## ファイル別の更新ルール

### flow-standard.md

**変更してよいもの:**
- フロー図のノード・矢印（業務ステップの追加・削除・変更）
- 「出典」欄の仕様書バージョン番号・日付
- 制度概要の数値（支給額・年齢・期限等）
- 標準仕様書が定める庁内連携の範囲

**変更してはいけないもの:**
- `gap-notes.md` に属する内容（システムと現実の差分）
- `incident-catalog/` への参照リンク（断ち切らない）

### gap-notes.md

**追記してよいもの:**
- 新たに発生したシステムと現実の差分
- 仕様書改版によって**解消されたギャップ**（`→ 解消済 [日付]` と記録して残す）
- 補正予算・事務連絡等、仕様書外の制度追加

**削除してはいけないもの:**
- 未解消のギャップ
- 過去のギャップ記録（履歴として残す）

### incident-catalog/

**追記してよいもの:**
- 新たに発生したインシデントパターン
- 全国事例の追加

**変更に慎重を要するもの:**
- `severity`（深刻度の格下げは別PRで議論）
- `related_journeys` / `related_workflows`（リポジトリ管理者のレビュー必須）

## Issueテンプレート（更新通知用）

```
タイトル: [更新通知] [業務名] [変更内容の概要]

## 変更の種類
- [ ] Type 1: 標準仕様書の改版（計画的）
- [ ] Type 2: 法改正・通知・補正予算（不定期）

## 変更内容
（省庁名・文書名・公開日・変更の概要）

## トリアージ結果
- [ ] A象限（フロー構造変更あり・判断ルール変更あり）
- [ ] B象限（フロー構造変更あり・判断ルール変更なし）
- [ ] C象限（フロー構造変更なし・判断ルール変更あり）
- [ ] D象限（参照更新のみ）

## 影響ファイル（候補）
- [ ] workflows/_standardized-20/xxx/flow-standard.md
- [ ] workflows/_standardized-20/xxx/gap-notes.md
- [ ] journeys/xxx.md
- [ ] incident-catalog/INC-xxx.md

## 優先度
- [ ] 即時  - [ ] 1週間以内  - [ ] 次回定期更新  - [ ] 月次確認
```

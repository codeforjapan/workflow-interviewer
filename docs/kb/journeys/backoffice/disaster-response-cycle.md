# 災害対応サイクル 🔄 O5

> **このファイルの役割**
> 自治体組織が大規模災害発生時〜復旧・復興期にわたって経験する「業務サイクル」を可視化する。
> 通常の年度サイクル（O1）が停止・変形する中で、どの業務がどう連鎖するかを示す。
> 住民の被災ジャーニー（`journeys/disaster-and-recovery.md`）の**行政側の対応サイクル**として位置づける。

---

## サイクル概要

災害対応は「非日常の業務」ではなく「通常業務が変形した形で継続する」状態である。発災から復興完了まで、自治体職員は通常業務と災害対応業務を並行して担い続ける。このサイクルはフェーズで区切られるが、実態はフェーズが重なり合いながら進む。

**災害対応サイクルの特徴:**
- 「いつ始まるかわからない」非定期サイクル
- 平時の組織体制・権限委任が一時的に変更される
- 応援職員・ボランティア・外部支援機関との調整が必要
- 通常業務（年度サイクルO1）が停止できないことが多い（住民票・税・保険料は続く）

---

## タイムライン

| フェーズ | 時期（目安） | 主な業務 | 発生するリスク・穴 |
|---|---|---|---|
| **初動・緊急対応** | 発災〜72時間 | 災害対策本部設置・避難所開設・安否確認・情報収集 | 庁内の意思決定指揮系統の混乱・担当者の被災 |
| **応急対応** | 発災後〜2週間 | 避難所運営・罹災証明書受付・応援職員の受入 | 住民対応ピークと職員疲弊の同期・情報の断絶 |
| **被害認定・証明書発行** | 発災後1〜4週間 | 被害認定調査・罹災証明書発行・不服申立対応 | 調査員不足・発行遅延・認定基準の地域差 |
| **支援制度の受付** | 発災後〜3か月 | 生活再建支援金・義援金・保険料減免等の受付 | 申請窓口の混雑・申請期限の管理・案内漏れ |
| **仮設住宅・移転対応** | 発災後〜6か月 | 仮設住宅の設置・入居・みなし仮設の調整 | 住民票の所在と実態の乖離・介護・福祉サービスの継続 |
| **復旧・復興** | 発災後〜数年 | インフラ復旧・住宅再建支援・コミュニティ再建 | 長期避難の孤立・財政負担・人口流出 |

## 詳細フロー

### フェーズ1: 初動・緊急対応（発災〜72時間）

```
災害発生（気象情報・震度情報等）
  │
  ├─ 首長の判断 → 避難指示・勧告の発令
  │   （市区町村の独自判断 or 都道府県知事・国の指示に基づく）
  │
  ├─ 災害対策本部の設置
  │   ├─ 本部長: 市区町村長
  │   ├─ 各部署の担当者の召集（休日・深夜でも）
  │   └─ 応急対応マニュアルの発動
  │
  ├─ 避難所の開設
  │   ├─ 指定緊急避難場所 → 指定避難所の順に開設
  │   ├─ 鍵・設備の管理担当者の確認
  │   └─ 避難所担当職員の派遣
  │       ⚠️ 担当職員自身が被災している場合の代替要員が未定のことがある
  │
  ├─ 安否確認・行方不明者情報収集
  │   ├─ 避難者名簿の作成
  │   ├─ 避難行動要支援者名簿との照合
  │   │   ⚠️ 名簿を持っている職員・組織が不在の場合に機能しない → INC-D01
  │   └─ 消防・警察・自衛隊への情報共有
  │
  └─ 都道府県・国への報告
      ├─ 被害報告（第1報・逐次更新）
      └─ 応援要請（応援協定・自衛隊派遣要請等）
```

### フェーズ2: 応急対応（発災後〜2週間）

```
避難所運営の本格化
  │
  ├─ 避難者の把握・名簿管理
  │   ├─ 全住民の安否確認（特に要配慮者）
  │   ├─ 避難所外避難者（車中泊・縁故避難）の把握
  │   │   ⚠️ 避難所外は支援物資・情報が届かない
  │   └─ 福祉避難所への移送調整（要配慮者）
  │       ⚠️ 福祉避難所の受け入れ容量・事前協定の機能確認 → INC-D02
  │
  ├─ 物資の調達・配給
  │   ├─ プッシュ型支援（国・都道府県から自動送付）の受入調整
  │   ├─ 避難所への配送ルート確保
  │   └─ 避難所外・仮設住宅への配給
  │
  ├─ 応援職員の受入
  │   ├─ 都道府県・他市区町村からの応援職員（LGWAN・応援協定）
  │   ├─ 役割分担・指揮系統の整理
  │   └─ 罹災証明書受付・被害認定調査チームの編成
  │
  └─ 通常業務との並行
      ⚠️ 住民票・税・保険料・介護・保育等の通常業務は止められない
      → 応援職員で分担するか、最低限の業務以外を一時停止の判断が必要
```

### フェーズ3: 被害認定・証明書発行（発災後1〜4週間）

```
罹災証明書申請の受付開始
  │
  ├─ 被害認定調査（外観調査・内部調査）
  │   ├─ 調査員の確保（本庁・応援・民間委託等）
  │   │   ⚠️ 大規模災害では調査員が圧倒的に不足
  │   ├─ 内閣府「被害認定基準運用指針」に基づく判定
  │   └─ AI・ドローン等を活用した外観調査の試み（一部自治体）
  │
  ├─ 罹災証明書の発行
  │   ├─ 区分決定（全壊/大規模半壊/中規模半壊/半壊/準半壊/一部損壊）
  │   └─ 交付（窓口・郵送・電子申請）
  │
  └─ 不服申立（第2次調査）の受付
      ├─ 申立書の受付
      ├─ 再調査の実施（内部担当者または外部専門家）
      └─ 区分変更の判定
          ⚠️ 不服申立の存在を知らない住民が多い → 交付時の説明が重要
```

### フェーズ4: 支援制度の受付（発災後〜3か月）

```
罹災証明書の発行が進む中で
  │
  ├─ 被災者生活再建支援金の受付
  │   ├─ 都道府県の受付窓口と連携
  │   ├─ 申請書類の確認・取りまとめ
  │   └─ 期限管理（発災から13か月以内が多い）
  │
  ├─ 義援金の配分
  │   ├─ 義援金配分委員会の設置
  │   ├─ 配分基準の決定（罹災証明書区分に応じた配分額）
  │   └─ 口座振込の実施
  │       ⚠️ 配分完了まで数か月〜1年以上かかることがある
  │
  ├─ 各種保険料・税の減免受付
  │   ├─ 国保・介護保険料の減免申請受付
  │   ├─ 固定資産税の減免申請受付
  │   └─ 個人住民税の減免処理
  │       ⚠️ 申請しないと自動減免されない制度が多い
  │       → 「申請していない被災者へのプッシュ型通知」が課題
  │
  └─ 社会福祉協議会との連携
      ├─ 生活福祉資金（緊急小口・総合支援）の案内
      └─ 災害ボランティアセンターの運営支援
```

---

## 穴の連鎖

```
発災直後の情報混乱
  └→ 被害状況が正確に把握できない
       └→ 被害認定調査が遅れる
            └→ 罹災証明書の発行が遅れる
                 └→ 各種支援制度の申請ができない
                      └→ 住民の生活再建が遅れる（数か月〜1年単位）

職員の被災・疲弊
  └→ 通常業務と災害対応の並行が困難
       └→ 通常の住民サービス（住民票・税・保険等）が劣化
            └→ 被災していない住民の不満・混乱

応援職員の指揮系統未整備
  └→ 役割分担が不明確で二重対応・抜け漏れが発生
       └→ 被災者対応の品質が低下

義援金の配分遅延
  └→ 被災者が「支援が来ない」と感じる
       └→ 行政不信・メディア批判が発生
```

## 行政が整備すべき事前準備（平時の課題）

| 課題 | 準備内容 | 担当部署 |
|---|---|---|
| 個別避難計画の未整備 | 要配慮者ごとに「誰が・どこへ・どの手段で」を計画 | 防災担当・福祉担当 |
| 罹災証明書発行体制 | 被害認定調査員のリスト・応援協定・AI活用 | 建設・住宅課・防災担当 |
| 義援金配分基準の事前設計 | 過去の事例を参考に配分委員会の設計 | 総務・福祉担当 |
| 応援職員受入マニュアル | 役割分担・指揮系統・宿舎確保等 | 総務・人事担当 |
| 通常業務の最低限継続計画 | BCP（業務継続計画）の整備・定期訓練 | 全庁 |
| 社協との事前連携 | 災害ボランティアセンター立ち上げの訓練 | 福祉担当・社協 |

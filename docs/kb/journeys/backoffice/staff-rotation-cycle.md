# 人事異動サイクル 🔄 O3

> **このファイルの役割**
> 3月内示から4月着任にかけての組織的な人員交代と、それに伴う引き継ぎ・知識継承の構造を可視化する。
> ベテランが持つ「この時期に何をしておかないと翌年度に何が壊れるか」という暗黙知を形式知化する。

---

## サイクル概要

自治体の人事異動は原則として年1回・4月1日付で実施される。1月〜3月の約3か月間に、異動希望調査・ポスト検討・内示・引き継ぎが集中し、その間に年度末業務（予算執行・決算準備）も重なる。

**なぜ問題になるか**: 引き継ぎ期間が実質2〜3週間に圧縮される中で、業務の手順だけでなく「なぜそうするのか」という背景知識・関係機関との関係性・暗黙のルールが十分に伝わらないまま4月を迎えることが常態化している。

---

## タイムライン

| 時期 | 主な業務 | 発生するリスク・穴 |
|---|---|---|
| **1月** | 異動希望調査（自己申告） | 申告内容が実際の配置と乖離し、モチベーション低下 |
| **2月上旬** | 課長・部長レベルの内々示・調整 | 水面下の調整が属人的で組織的な配置基準が不明確 |
| **2月下旬〜3月上旬** | 内示（本人への通知） | 内示から引き継ぎ開始まで2〜3週間しかない |
| **3月中旬〜下旬** | 引き継ぎ書作成・引き継ぎ実施 | 形式的な引き継ぎ書のみで暗黙知が伝わらない |
| **3月末** | 年度末業務・会計締め・人事異動最終調整 | 年度末業務（O1交差）と引き継ぎが並走し、どちらも不完全になる |
| **4月1日** | 辞令交付・着任 | 着任初日から住民対応・業務処理が始まる |
| **4月〜5月** | 新任者の業務習熟期間 | 業務品質が最も低下する時期。窓口ミス・処理遅延が増加 |
| **5月〜6月** | 会計年度任用職員の更新・採用 | 正規職員の異動後に支援職員も入れ替わり、引き継ぎ二重苦 |

---

## 穴の連鎖

```
[3月末] 年度末業務の繁忙で引き継ぎ時間が確保できない（O1交差）
  └→ [4月] 着任者が業務の背景・経緯を知らない状態でスタート
       └→ [4〜5月] 窓口対応ミス・処理遅延・関係機関との連携断絶
            ├→ 住民への不利益（手続き誤案内、期限超過） → incident-catalog
            └→ [翌年度] 事業の継続性が失われ、同じミスを繰り返す
```

```
[3月内示後] システムアクセス権限・パスワード管理が曖昧なまま引き継ぎ
  └→ [4月] 前任者のアカウントが残存 or 後任者が権限なしで業務開始
       └→ セキュリティリスク・業務停滞・不正アクセスリスク
```

```
[4月] 福祉・介護担当の異動により、個人の支援状況が引き継がれない
  └→ 支援継続中のケース（要介護・生活保護等）への連絡漏れ
       └→ サービス中断・住民の権利侵害 → INC-004, INC-007参照
```

## 引き継ぎの3層構造

引き継ぎには以下の3つの層があるが、実際には1層目しか伝わらないことが多い。

| 層 | 内容 | 伝わりやすさ | 引き継ぎ手段 |
|---|---|---|---|
| **1層: 手順知識** | 「この業務の手順はこうだ」 | 伝わりやすい | 引き継ぎ書・マニュアル |
| **2層: 背景知識** | 「なぜこの手順か」「例外対応の判断基準」 | 伝わりにくい | 口頭・メモ・同席 |
| **3層: 関係知識** | 「誰に聞けばわかるか」「関係機関との関係性」 | ほぼ伝わらない | 内部KB（Notion等）での管理推奨 |

> **3層目（関係知識）** は、このGitHubリポジトリではなく内部KB（Notion）で管理する。
> 担当者名・関係機関の連絡先・交渉の経緯などは `CONTRIBUTING_INTERNAL.md` 参照。

## 担当者向け: 引き継ぎチェックリスト

### 引き継ぐ側（前任者）

```
□ 1. 進行中の案件一覧（締切・関係者付き）を作成する
□ 2. 例外対応の事例と判断根拠を書き残す
□ 3. システムアクセス権限の移管手続きを完了する
□ 4. 関係機関の担当者情報を後任者に紹介する（内部KBへの記録）
□ 5. 「このタイミングに必ずやること」リスト（年間カレンダー）を渡す
□ 6. 未解決の課題・懸案事項を明示する（押し付けではなく、状況説明として）
```

### 引き継ぐ側（後任者）

```
□ 1. 最初の1週間で進行中の案件を全件把握する
□ 2. わからないことを前任者・同僚に積極的に聞く（恥ずかしくない）
□ 3. 関係機関に「新しく担当になった」と挨拶連絡する
□ 4. 年間カレンダーで「次の締切」を確認する
□ 5. 業務フロー（このリポジトリ）で標準的な手順を確認する
```

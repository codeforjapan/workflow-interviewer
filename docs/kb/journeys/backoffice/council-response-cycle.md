# 議会対応サイクル（定例会4回＋臨時会） 🏛️ O2

> **このファイルの役割**
> 1つの定例会サイクル（約2か月）で、全庁の職員が経験する議会対応業務を示す。
> 個々の業務フローは `workflows/` を参照。業務間の「穴」は `incident-catalog/` を参照。
> **議会対応経験の浅い職員は「このファイルから読み始める」ことを推奨する。**

---

## ジャーニーの全体像

1つの定例会サイクルは、議会開会6週間前から、閉会後の決議執行まで、約2か月間の業務連鎖。

```
【1つの定例会サイクル（約2か月）】

議会開会6週間前
  │
  ├─ 議案提出期限
  │   └─ 「この定例会で提出する議案」の最終期限
  │       ├─ 予算案・条例改正案・人事案など
  │       └─ 提出期限を1日過ぎると定例会に間に合わない
  │
  ├─ 法制審査（条例改正の場合）
  │   └─ 議案としてのテクニカルチェック
  │       └─ 本当に「条例」で良いのか、施行日は正しいか
  │
  └─ 全員協議会での概要説明（定例会開会4週間前）
      └─ 議員全員に「これから何が議題になるか」をざっくり説明
      └─ 異議なし → OK、異議あり → 協議開始


議会開会2週間前
  │
  ├─ 議案の最終校正
  │   └─ 誤字脱字がないか、法律上の誤りがないか
  │
  └─ 議案を印刷して全議員に配布
      └─ ここから議員が「事前に資料を読む期間」開始


議会開会（定例会当日以降）
  │
  ├─ 【初日】本会議：市長あいさつ・議案の説明
  │   └─ 市長から「これからこんなことをします」という提案
  │
  ├─ 【2日目以降】一般質問
  │   └─ 議員が「市長、これどうなってるの？」という質問
  │   └─ 市役所側が「答弁書」で答える（1議員あたり15分程度）
  │
  ├─ 【質問後】各委員会（常任委員会・特別委員会）
  │   ├─ 総務委員会・福祉文教委員会など（自治体で異なる）
  │   ├─ 各委員会が担当の議案について詳しく審査
  │   ├─ 委員から「これ何ですか？」という質問 → 担当課が説明
  │   └─ 質疑が終わったら採決
  │
  └─ 【最終日】本会議：採決（議案が可決・否決される）


議会閉会後
  │
  ├─ 決議事項の執行
  │   └─ 「この条例は4月1日から施行」などを実行に移す
  │
  ├─ 陳情への処理（陳情が上がっていた場合）
  │   └─ 「採択された陳情」は関係課で検討を開始
  │
  └─ 次の定例会に向けた宿題整理
      └─ 委員会で指摘されたことを「次はこうする」と整理
      └─ ← この整理が放置されると穴になる


    ↓
  約2か月後、次の定例会が来る
```

### 定例会のスケジュール（標準）

| 定例会 | 月 | 標準的な開会日 | 議案提出期限 | 主な内容 |
|---|---|---|---|---|
| **第1回** | 3月 | 3月初週 | 1月下旬 | **新年度当初予算・条例** |
| **第2回** | 6月 | 6月初〜中旬 | 4月中旬 | **前年度決算認定・条例** |
| **第3回** | 9月 | 9月初〜中旬 | 7月下旬 | **補正予算・決算認定** |
| **第4回** | 12月 | 12月初旬 | 10月下旬 | **補正予算・条例** |

**臨時会**は必要に応じて開かれる（人事案件、緊急対応など）

## 業務連鎖マトリクス：1つの定例会サイクルの全体像

### フェーズ1：議案提出前（6週間前）

| 業務 | 担当課 | 期限 | リスク |
|---|---|---|---|
| 議案の企画・原案作成 | 全課 | 決まっていない（担当課の都合） | 遅延すると議案提出期限に間に合わない |
| 法制審査（条例案の場合） | 総務課・法制審査係 | 議案提出期限の1週間前 | 指摘が多いと修正に時間がかかる |
| 全員協議会の資料準備 | 議会事務局・各課 | 定例会開会4週間前 | 概要がわかりにくいと議員から異議 |

### フェーズ2：議案提出〜議案配布（3〜2週間前）

| 業務 | 担当課 | 期限 | リスク |
|---|---|---|---|
| 議案提出 | 全課 | 議案提出期限厳守 | 1時間遅れても次の定例会に延期される |
| 議案の最終校正 | 議会事務局・各課 | 提出から5日以内 | 誤字脱字が議案に残ると指摘される |
| 議案の印刷・配布 | 議会事務局 | 定例会開会2週間前 | 印刷トラブルで配布遅延 → 議員から苦情 |

### フェーズ3：一般質問対応（開会1週間前〜当日）

| 業務 | 担当課 | 期限 | リスク |
|---|---|---|---|
| 一般質問の通告受付 | 議会事務局 | 開会3日前（典型） | **通告が遅く来ると、答弁書作成が深夜になる** 🔴 |
| 答弁書の作成・提出 | 全課 | 通告受付から48時間以内（標準） | 時間が足りず、答弁内容が曖昧になる |
| 本会議での一般質問 | 全課（答弁担当者） | 定例会中 | 想定外の追加質問に答えられない |

### フェーズ4：委員会質疑（開会1週間後〜2週間後）

| 業務 | 担当課 | 期限 | リスク |
|---|---|---|---|
| 委員会資料の追加準備 | 各課 | 開会から1週間以内 | 「この事業、詳しく説明して」という要求に間に合わない |
| 委員会での説明・質疑対応 | 各課（説明担当者） | 委員会の日時 | 担当課長が説明を間違える、または担当者を同席させていない |
| 委員会での採決 | 議会 | 委員会の最終日 | **採決後の「宿題」が記録されない** 🔴 |

### フェーズ5：本会議採決〜決議執行（閉会後）

| 業務 | 担当課 | 期限 | リスク |
|---|---|---|---|
| 議案の採決 | 議会 | 定例会最終日 | 想定と違う結果になることも（否決など） |
| 可決された議案の施行準備 | 各課 | 条例は施行日までに | 「4月1日施行」という議決が「できていない」まま4月になる |
| 陳情への回答 | 各課 | 陳情の判定から30日以内（標準） | **「採択された陳情」の検討が放置される** 🔴 |
| 委員会での指摘事項の整理 | 各課 | 委員会後1週間以内 | **「宿題」が記録されず、来月も同じ指摘を受ける** 🔴 |

## 穴の連鎖マップ

議会対応サイクルで発生しうるインシデントを、発生箇所と連鎖先で示す。

```
一般質問通告（開会3日前）
  │
  └─[穴A] 一般質問の通告が直前に来る
         ├─ 標準は「3日前」だが、実際には「開会当日朝」に来ることもある
         ├─ 各課が答弁書を急いで作成 → 深夜残業
         ├─ 答弁内容が曖昧になる、誤字脱字が残る
         ├─ 本会議で「その答弁、間違ってます」と指摘される
         └─ incident-catalog/general-question-rush.md

         ⚠️ 暗黙の文化：「答弁案は議員の意図を汲んで作るべき」
             → つまり「ネガティブな回答をするな、ポジティブに言え」
             → 結果、事実と異なる答弁をしてしまうリスク

委員会質疑（開会1週間後）
  │
  ├─[穴B] 委員から「これ、詳しく説明して」と言われても、
  │       追加資料がない
  │       ├─ 「今週の委員会で説明します」と言っておいて、
  │       │   資料が間に合わない
  │       ├─ 委員会当日に「申し訳ございません、来週提出します」
  │       └─ 信頼度低下
  │
  └─[穴C] 委員会での指摘事項が「次の議会まで放置」される
          ├─ 「今月は 〇〇 という指摘を受けました」と記録されず
          ├─ 次の議会で同じ質問を受ける → 「あ、その指摘覚えてません」
          ├─ 議員から「毎回同じ質問させるな」という怒り
          └─ incident-catalog/council-todo-follow-up.md

本会議採決（定例会最終日）
  │
  └─[穴D] 採決結果が想定と違う（否決など）
          ├─ 「可決されるはず」と思っていた議案が否決される
          ├─ 「え、なぜ？」という状態で決議執行の準備ができていない
          ├─ 条例は施行予定日が決まっているのに、可決されていない
          └─ incident-catalog/council-unexpected-result.md

陳情処理（閉会後）
  │
  └─[穴E] 「採択された陳情」の検討が始まらない
          ├─ 「陳情者への回答期限は30日」という規則があるはず
          ├─ だが「採択されたから何かするのか」の理解がない
          ├─ 期限が来たら「いや、この陳情、実は難しい」と回答
          ├─ 陳情者から苦情がくる
          └─ incident-catalog/petition-follow-up-gap.md

条例施行準備（採決後）
  │
  └─[穴F] 「〇月〇日から施行」という採決をしたのに、
          準備ができていない
          ├─ システム改修が必要なのに、
          │   委員会で「これいつから？」と聞かれて「え、〇月？」
          ├─ 案内票の準備ができていない
          ├─ 職員研修がされていない
          ├─ 〇月初日に「あ、準備忘れてた」という状態に
          └─ incident-catalog/ordinance-enforcement-gap.md

```

## 担当者向けチェックフロー

議会対応経験の浅い職員が、1つの定例会サイクルで確認すべきことをチェックリスト化した。

### **フェーズ1：議案企画〜提出前（6週間前）**

```
□ 1. 「この定例会で提出する議案」の一覧は確定したか
       └─ 「提出するつもり」と「実際に提出する」は違う
       → 議会事務局に「こういう議案を考えています」と相談

□ 2. 条例改正が必要な場合、法制審査の期間は確保されているか
       └─ 「条例改正」と決まったら、すぐに原案を作成
       → 法制審査係に提出（提出期限の1週間前を目安）
       → 修正指摘が来たら、修正に数日かかることも
       → incident-catalog/ordinance-review-delay.md を参照

□ 3. 全員協議会での説明資料は、わかりやすく書かれているか
       └─ 議員が初めて見る資料になる
       → 「その議案で何が変わるのか」が3行で説明できるか
       → 背景・目的・具体的な内容の順序で書く

□ 4. 「議案提出期限」は正確に把握されているか
       └─ 自治体ごとに異なる（典型的には3週間前）
       → 提出期限を過ぎると「次の定例会」に延期される
       → 重要な議案なら「提出期限の1か月前」くらいから進める
```

### **フェーズ2：議案提出〜配布前（2週間前）**

```
□ 1. 提出した議案に誤字脱字がないか（最終校正）
       └─ 議会事務局が確認する期間は「提出から5日以内」
       └─ 誤字が見つかると、修正版を再提出
       → 修正に時間がかかると議案配布が遅延

□ 2. 法律上の誤りがないか（法制審査）
       └─ 「条例の構文」は法律で決まっている
       → 施行日の書き方、公布日の計算など
       → 総務課・法制審査係からの指摘に対応する
       → incident-catalog/ordinance-review-delay.md 参照

□ 3. 議案配布（定例会開会2週間前）が予定通りか
       └─ 議会事務局が印刷・配布する
       → 印刷トラブルで遅延することもある
       → 配布が遅れると「議員が資料を読む時間がない」というクレーム
```

### **フェーズ3：一般質問対応（開会1週間前〜当日）**

```
□ 1. 一般質問の通告期限は正確に把握しているか
       └─ 標準は「開会3日前」（自治体による）
       → 通告が遅く来ると、答弁書作成が間に合わない
       → 「通告が〇月〇日」と担当課に伝える

□ 2. 想定される質問事項は事前に整理しているか
       └─ 「議員は〇〇について聞きそうだ」という予測
       → あらかじめ答弁案を作成しておく
       → 通告が来たら「あ、その質問ですね」ですぐ対応できる
       → incident-catalog/general-question-rush.md 参照

□ 3. 答弁書の誤字脱字チェックが完了したか
       └─ 本会議で読み上げる公式文書
       → 誤字脱字があると、議員から指摘される
       → 誤字脱字のほか「事実と異なる」がないか確認

□ 4. 本会議で答弁する職員（課長など）はリハーサルしているか
       └─ 「紙を読み上げるだけ」では不十分
       → 追加質問に備えて、事実を頭に入れる
       → ウソの答弁をしないよう、事前に経営層に相談
       → incident-catalog/general-question-false-answer.md 参照

□ 5. 部長級の職員は「全般答弁」をしているか
       └─ 「自分の担当以外の質問」にも答えることがある
       → 他課の業務を最低限理解しておく
       → 「調べます」では答弁にならない
```

### **フェーズ4：委員会質疑対応（開会1週間後）**

```
□ 1. 委員会に出席する説明者の準備は整っているか
       └─ 課長が出席する、または担当係長が同席する
       → 「部長だけ」では、細部の質問に答えられない
       → 複雑な議案なら「実務担当者」を同席させる

□ 2. 「この議案について聞かれそうなことは？」をリストアップしたか
       └─ 委員会の質疑は「細部の質問」が多い
       → 「なぜこの予算？」「前年と比べて？」「効果は？」
       → あらかじめ想定して、説明資料を用意
       → 委員から「詳しく説明して」と言われても、
         資料がある → すぐ提出できる

□ 3. 予算議案の場合、積算根拠は明確か
       └─ 「なぜこの金額？」という質問は必ず出る
       → 「〇人 × ￥〇 = ￥〇〇」という計算式が説明できるか
       → 根拠が曖昧な予算は質疑が長引く

□ 4. 人事案件の場合、経歴・能力は説明できるか
       └─ 「なぜこの人が昇進？」という質問は必ず出る
       → 「適正に選考しました」では答弁にならない
       → 経歴、実績、能力評価を説明できるよう準備
       → incident-catalog/personnel-case-explanation.md 参照

□ 5. 委員からの質問を記録しているか
       └─ 「今月の指摘」と「来月の指摘」が別々になっていないか
       → 委員会の「議事録」には質疑が記録される
       → その議事録を読んで、来月に「同じ質問をされない」ようにする
       → incident-catalog/council-todo-follow-up.md 参照
```

### **フェーズ5：本会議採決〜決議執行（閉会後）**

```
□ 1. 採決結果は想定通りだったか
       └─ 「可決」だと思っていたが「否決」されることもある
       → 否決の場合、その理由を把握する
       → 「なぜ否決されたのか」を分析して、
         次の議会でどう対応するか考える

□ 2. 可決された議案の「施行日」は記録されたか（条例の場合）
       └─ 「〇月〇日から施行」という採決がされている
       → その日までに「システム改修」「職員研修」「案内票作成」を終わらせる
       → 施行日が来たのに準備ができていない、という事態を避ける
       → incident-catalog/ordinance-enforcement-gap.md 参照

□ 3. 陳情が「採択」された場合、その後のプロセスは確認したか
       └─ 「採択された陳情」には「30日以内に回答する」というルールがある
       → 「陳情が採択=市が実行する」ではない
       → 「採択された陳情について、市は以下の通り対応します」という
         実行計画を作成する必要がある
       → これを「放置」すると、陳情者から苦情が来る
       → incident-catalog/petition-follow-up-gap.md 参照

□ 4. 委員会で指摘された事項は、どの課が何をするのか確認したか
       └─ 例：「〇〇サービスを今後どうするのか、次回報告して」
       → この「宿題」を記録して、次の委員会での報告準備を始める
       → もし放置すると、次の委員会で「あ、まだ検討してません」
       → 議員から「毎回同じ質問させるな」という批判
       → incident-catalog/council-todo-follow-up.md 参照

□ 5. 新しい条例は施行準備が完了したか
       └─ 「令和〇年〇月〇日から施行」と議決された
       → その前までに：
         ├─ システム改修
         ├─ 案内票・申請様式の作成
         ├─ 職員研修
         ├─ 住民周知
         └─ すべて完了しているか確認
       → 施行日当日に「あ、忘れた」が起きないように
       → incident-catalog/ordinance-enforcement-gap.md 参照
```

## 住民ジャーニー × 議会対応サイクル の交点

O2（議会対応）が他のサイクルと交差しやすい時期と、そのリスク：

### **9月定例会：O1 × O2（年度サイクル × 議会対応）**

| 時期 | O1で起きること | O2で起きること | クロス時のリスク |
|---|---|---|---|
| **9月** | 前年度決算認定・補正予算編成開始 | 9月定例会（決算認定・補正予算議決） | 同じ「補正予算」について、委員会での質疑が厳しい |

**具体的な穴**：
- 決算認定議会で「去年、なぜこれだけ不用額が出たのか」と質問される
- 同時に「今年の補正予算、これ何ですか？」と質問される
- 両方に同時に答える = 財政課が疲弊する

---

### **3月定例会：O1 × O2 × O3（年度サイクル × 議会対応 × 人事異動）**

| 時期 | O1で起きること | O2で起きること | O3で起きること | クロス時のリスク |
|---|---|---|---|---|
| **3月** | 新年度予算議決・年度末執行 | 3月定例会（予算特別委員会） | 人事異動内示 | 答弁者が異動することが決まり、引き継ぎができていない状態で答弁 |

**具体的な穴**：
- 予算特別委員会で「この事業、去年と何が変わったのか」と聞かれる
- その説明者が「3月末で異動が決まった」ので、詳しくない
- 答弁内容が曖昧 → 委員から指摘 → 次の定例会で同じ質問をされる

## 一般質問への答弁書作成：よくある穴

### 穴A：答弁が「曖昧」になる

```
【悪い例】
質問：「〇〇サービスの利用者は増えているのか？」
答弁：「市民のニーズは高いと考えており、今後も推進していきます」
→ 「増えているのか、増えていないのか、どっち？」と追加質問される
→ 翌週の本会議で「きちんと数字で答えてください」と指摘される

【良い例】
質問：「〇〇サービスの利用者は増えているのか？」
答弁：「令和4年度は100人、令和5年度は120人、令和6年度は145人となっており、
      毎年約20%の増加傾向にあります。
      理由としては、制度周知の強化と利用要件の緩和が考えられます」
→ 「納得」
```

**対策**：答弁書には「数字」「理由」「今後の方針」を必ず盛り込む

### 穴B：答弁が「ウソ」になる

```
【危険な例】
質問：「この予算、本当に効果があるのか？」
答弁：「はい、十分な効果があります」
→ 後になって「実は効果がない」という監査指摘が出る
→ 議員から「議会で「効果がある」と言ったのに、ウソだったのか」と怒られる

【安全な例】
質問：「この予算、本当に効果があるのか？」
答弁：「初年度のため、確実な効果測定は来年度以降の実績を踏まえて
      行う予定です。今年度は利用者数・満足度調査を実施し、
      改善に向けた基礎データを集める方針です」
→ 「そういうアプローチなら仕方ない」と納得される
→ 後で「効果がない」となっても「来年度改善する」という方針が建てやすい
```

**対策**：確実でないことは「確認します」「来年度測定します」と正直に答える

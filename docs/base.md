---

## 📌 プロジェクトコア・ビジョン

### 「AIエージェント・インタビューアプリ」

**課題**: 自治体職員の暗黙知（業務ノウハウ）が個人にとどまり、マニュアル化できない

**ソリューション**: AIが職員にヒアリングを行い、**その場でリアルタイムに業務フロー図を生成**

**成果**: 業務の可視化・デジタル化・組織知化のためのツール

### 利用シーン

```
職員が話す
    ↓
AIが音声認識 + ヒアリング（LiveKitはつかえそうだけど、そこまでいらないかも）
    ↓
構造化データ抽出
    ↓
キャンバス上にフロー図がリアルタイム生成（https://reactflow.dev/とかtldrawとか？）
    ↓
ファシリテータがその場で図を調整
    ↓
セッション完了時に完成形をファイル出力？
```

---

## 🎯 3フェーズのコンセプト設計

### 【3者構成モデル】

- **対象職員**: 業務の話し手
- **AIアバター**: インタビュアー（LiveKit Agents経由の音声対話、職員は音声でAIはテキストでもいいかも）
- **人間ファシリテータ**: 職員のサポート + 図解の手動調整。ハイクオリティで全部AIがやらなくてもいいという補助として考えてもらっていい

### 【心理的安全性の実装】

- 職員: 「適当に話しても大丈夫」という安心感
- 視覚的フィードバック: リアルタイムで話した内容が図になっていく楽しさ
- セキュリティ可視化: 「この会話は外部に流出しません」という信頼

### 【5フェーズのインタビューフロー】

| フェーズ                    | 目的             | AIの役割                           |
| --------------------------- | ---------------- | ---------------------------------- |
| **Phase 1: アイスブレイク** | 心理的安全性構築 | 親しみやすいトーンで自己紹介       |
| **Phase 2: 業務概要把握**   | 全体像確認       | 業務名、目的、根拠法令をヒアリング |
| **Phase 3: 深掘り**         | 分岐・例外抽出   | 「〇〇の場合は？」型の条件分岐質問 |
| **Phase 4: 検証・補完**     | 正確性確認       | 生成フロー図を提示し修正依頼       |
| **Phase 5: クローズ**       | セッション終了   | サマリー＋次回引き継ぎ事項確認     |

## 🏗️ システムアーキテクチャ（概要）

### 主要コンポーネント

```
【フロントエンド層】
  └─ React
     ├─ 対話ビュー
     ├─ Canvasビュー（リアルタイムフロー図）
     └─ Chat/Transcript（会話ログ表示）

【リアルタイム通信層】
  └─ LiveKit？ (OSS)
     ├─ LiveKit
     ├─ LiveKit Agents（AI エージェント実行環境）
     └─ WebSocket（リアルタイムデータ同期）

【AI エージェント層】
  └─ Typescript？
     ├─ STT プラグイン（Deepgram/Whisper）
     ├─ LLM（OpenAI GPT-4o / Claude）
     ├─ TTS プラグイン（OpenAI/VOICEVOX）
     └─ スキーマ抽出ロジック

【データ層】
  └─ PostgreSQL + pgvector
     ├─ 業務基本情報
     ├─ インタビューセッション
     └─ 構造化フロー（JSON）
```

### 3つの技術的メリット

1. **LiveKit セルフホスト**
   - 音声データが閉鎖環境内に留まる
   - LGWAN対応への道が開ける
   - AI エージェント（STT/LLM/TTS）が統一基盤で動作
2. **ReactFlow や tldraw による可視化**
   - Mermaid 等の構文ベースより自由度高い
   - ファシリテータが手動調整可能
3. **複数出力形式**
   - JSON: AI 自動化ツール（Dify/n8n）への連携
   - Markdown: 業務マニュアル化
